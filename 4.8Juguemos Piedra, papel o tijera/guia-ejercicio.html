Guía del ejercicio: piedra, papel o tijera

Archivos de referencia: index.html y js/app.js. Las líneas citadas corresponden a la versión guardada cuando se redactó esta guía.

En qué consistía el ejercicio

Se pedía un programa jugable en el que el ordenador elige al azar un número entre 1 y 3 (1 = piedra, 2 = papel, 3 = tijera). El usuario introduce también un número entre 1 y 3 tras ver las opciones y el texto tipo “Tu elección (1-3):”.

Si el valor no es válido (cualquier cosa que no sea exactamente 1, 2 o 3 como se pidió), debe mostrarse el mensaje: «Entiendo que no quieres jugar. Adiós».

Si es válido, se debe mostrar el resultado con el formato del enunciado, por ejemplo: «Yo papel y tú tijera. ¡Has ganado!», usando las palabras piedra/papel/tijera y el cierre ¡He ganado! o ¡Has ganado! según corresponda.

js/app.js — qué hace cada parte (por líneas)

Línea(s) Contenido
1–5 Objeto nombres: convierte 1, 2 o 3 en las cadenas "piedra", "papel", "tijera".
7–10 esValido(valor): convierte a número, exige entero y que sea 1, 2 o 3.
12–18 ganaUsuario(usuario, maquina): devuelve true si el usuario gana según reglas clásicas (piedra a tijera, etc.).
20–33 textoResultado(maquina, usuario): arma el mensaje. Empate texto plano; victoria del usuario incluye HTML para pintar ¡Has ganado! en rojo; resto, texto plano (por ejemplo «He ganado»). Devuelve { esHtml, texto }.
35–37 Referencias al DOM: campo de elección, párrafo de mensaje, botón jugar (por id).
39–55 Al hacer clic: lee value del input; si no válido, mensaje de Adiós y return. Si válido, convierte a número, sortea maquina con Math.random(), obtiene resultado; si lleva HTML usa innerHTML, si no textContent.

El sorteo usa Math.floor(Math.random() * 3) + 1 para obtener 1, 2 o 3 con probabilidad uniforme aproximada.

15 preguntas tipo test (A / B / C) — respuestas ocultas

Haz el quiz sin abrir los <details>. Cuando termines, abre cada bloque para comprobar.

1. ¿Qué representa el número 2 en el juego?

  • A) Piedra
  • B) Papel
  • C) Tijera
Respuesta

B. 2 corresponde a «papel» (como en el enunciado y en el objeto nombres).

2. ¿Qué función de JavaScript sirve para obtener un entero aleatorio entre 1 y 3 inclusives?

  • A) Math.random() * 3 solamente
  • B) Math.floor(Math.random() * 3) + 1
  • C) Math.ceil(Math.random() * 2)
Respuesta

B. Math.random() da [0,1); al multiplicar por 3 y usar floor se obtiene 0, 1 o 2; al sumar 1 queda 1, 2 o 3.

3. Si el usuario deja el campo vacío y pulsa Jugar, ¿qué ocurre?

  • A) Se elige piedra por defecto
  • B) Aparece el mensaje «Entiendo que no quieres jugar. Adiós»
  • C) El programa se bloquea
Respuesta

B. El valor no es un entero 1–3 válido; esValido falla y se muestra el mensaje del enunciado.

4. ¿Para qué sirve document.getElementById("eleccion")?

  • A) Para crear un nuevo input
  • B) Para enlazar el script con el <input id="eleccion"> del HTML
  • C) Para borrar el mensaje
Respuesta

B. Busca en el DOM el elemento con ese id y devuelve su referencia.

5. ¿Por qué el botón usa type="button"?

  • A) Para que sea más grande
  • B) Para que no intente enviar un formulario (no hay comportamiento submit accidental)
  • C) Porque es obligatorio en HTML5
Respuesta

B. Dentro de un formulario, submit enviaría; aquí solo dispara la lógica del juego.

6. Piedra gana a…

  • A) Papel
  • B) Tijera
  • C) Empate siempre
Respuesta

B. Piedra vence a tijera (las tijeras se rompen contra la piedra).

7. Tijera gana a…

  • A) Piedra
  • B) Papel
  • C) Nada
Respuesta

B. Tijera corta papel.

8. Si usuario y máquina sacan el mismo número, en este proyecto el mensaje incluye…

  • A) «¡He ganado!»
  • B) «¡Has ganado!»
  • C) «¡Empate!»
Respuesta

C. En textoResultado, cuando maquina === usuario se devuelve la cadena de empate.

9. ¿Qué propiedad se usa para volcar texto sin interpretar etiquetas HTML?

  • A) innerHTML
  • B) textContent
  • C) value del body
Respuesta

B. textContent trata todo como texto. innerHTML sí parsea HTML (se usa solo cuando hace falta el “Has ganado” en rojo).

10. El atributo defer en la etiqueta script…

  • A) Borra el script
  • B) Retrasa la ejecución hasta que el documento HTML esté parseado
  • C) Hace el script asíncrono sin orden
Respuesta

B. defer mantiene el orden con otros scripts diferidos y ejecuta tras el parseo del documento.

11. ¿Qué valida Number.isInteger(n) en este ejercicio?

  • A) Que el número sea mayor que 10
  • B) Que sea un entero (no 2.5 ni NaN)
  • C) Que sea texto
Respuesta

B. Evita aceptar decimales u otros valores que no sean enteros matemáticos.

12. El meta name="viewport" en el HTML sirve principalmente para…

  • A) Cambiar el color de fondo
  • B) Ajustar cómo se escala la página en móviles
  • C) Cargar Tailwind
Respuesta

B. Indica ancho y zoom inicial; sin él, muchos móviles muestran una versión “miniatura”.

13. ¿Dónde debe estar app.js para que lo cargue <script src="js/app.js">?

  • A) En la raíz del proyecto junto a index.html
  • B) Dentro de una carpeta js/ con el nombre app.js
  • C) Dentro de img/
Respuesta

B. La ruta relativa es carpeta js, archivo app.js.

14. En el mensaje de victoria del jugador, «¡Has ganado!» aparece en rojo porque…

  • A) Tailwind lo pinta solo
  • B) Se envuelve en HTML con estilo (color) y se inserta con innerHTML
  • C) El navegador colorea siempre las victorias
Respuesta

B. Esa parte se construye con un <strong style="color:#b91c1c"> dentro del string mostrado como HTML.

15. aria-live="polite" en el párrafo del mensaje ayuda a…

  • A) Hacer la fuente más grande
  • B) Anunciar cambios del texto a lectores de pantalla sin interrumpir de forma agresiva
  • C) Desactivar JavaScript
Respuesta

B. Es una pista de accesibilidad: cuando cambia el resultado, puede notificarse al usuario que usa lector de voz.
