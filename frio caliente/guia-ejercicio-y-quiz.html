Guía del ejercicio: Frío / Tibio / Caliente

En qué consistía el ejercicio

El ejercicio es un juego de adivinanza: el programa elige al cargar la página un número entero secreto entre 1 y 499 (inclusive). El usuario escribe un número en un campo, pulsa Adivinar y recibe pistas según la distancia al número secreto: frío (lejos), tibio (intermedio) o caliente (cerca), además de si su número es mayor o menor que el secreto. Si acierta, el fondo pasa a verde, se muestra el número, se deshabilita el input y se oculta el botón de envío.

Tecnologías usadas: HTML (estructura y formulario), CSS/Tailwind (presentación), JavaScript (lógica en js/app.js, enlazado al final del body).

Quiz: 15 preguntas tipo A / B / C

Las respuestas están dentro de <details>: no abras el desplegable hasta terminar cada pregunta (o al corregir).

1. ¿Qué hace event.preventDefault() en el envío del formulario?

  1. Borra el número del input.
  2. Evita el comportamiento por defecto (recargar la página al enviar).
  3. Genera un nuevo número secreto.
Respuesta

B. Sin preventDefault, el navegador enviaría el formulario de forma clásica y recargaría la página.

2. ¿Cuál es el rango del numeroSecreto en app.js?

  1. De 0 a 499 inclusive.
  2. De 1 a 499 inclusive.
  3. De 1 a 500 inclusive.
Respuesta

B. Math.floor(Math.random() * 499) + 1 da enteros entre 1 y 499.

3. ¿Qué distancia mínima corresponde a “Frío, frío”?

  1. 50 o más unidades de diferencia respecto al secreto.
  2. Menos de 15 unidades.
  3. Exactamente 50 siempre.
Respuesta

A. Si distancia >= 50 se usa el nivel frío.

4. ¿Qué pasa con el botón cuando el usuario acierta?

  1. Se deshabilita con disabled.
  2. Se oculta con display: none.
  3. Cambia el texto a “Fin”.
Respuesta

B. Se asigna boton.style.display = "none".

5. ¿Para qué sirve novalidate en el <form>?

  1. Para que el navegador no aplique su validación HTML5 y puedas controlar todo en JS.
  2. Para validar solo con CSS.
  3. Para desactivar el botón enviar.
Respuesta

A. Desactiva la validación nativa del formulario; la comprobación la hace revisar.

6. ¿Qué elemento del DOM muestra los mensajes “Frío…”, “Tibio…” o la victoria?

  1. El <label for="numero">.
  2. El elemento con id="resultado".
  3. El <header>.
Respuesta

B. El script usa resultado.textContent = ... sobre ese párrafo.

7. Si juegoTerminado es true, ¿qué ocurre al pulsar de nuevo Adivinar?

  1. Se cuenta otro intento y se comparan números.
  2. revisar sale al inicio sin cambiar nada.
  3. Se reinicia el número secreto.
Respuesta

B. El if (juegoTerminado) return; corta la función.

8. ¿Cómo se calcula la “distancia” al número secreto?

  1. valor - numeroSecreto sin valor absoluto.
  2. Math.abs(valor - numeroSecreto).
  3. Math.random() * valor.
Respuesta

B. El valor absoluto mide cercanía sin importar si el intento es mayor o menor.

9. ¿Qué atributo del label lo une al campo de número?

  1. name="numero"
  2. for="numero" enlazado con id="numero" del input.
  3. href="#numero"
Respuesta

B. El par for / id asocia label e input.

10. En aplicarFondo, ¿qué hace colores[estado] ?? "#000000"?

  1. Siempre pinta de negro.
  2. Si estado no tiene color en el objeto, usa negro como respaldo.
  3. Borra el fondo.
Respuesta

B. El operador ?? elige el valor de la derecha solo si la izquierda es null o undefined.

11. ¿Por qué el <script src="./js/app.js"> va al final del body?

  1. Por obligación del estándar HTML.
  2. Para que el DOM (ids como numero) exista antes de ejecutar getElementById.
  3. Para cargar más rápido las imágenes.
Respuesta

B. Así los elementos ya están en el documento cuando corre el script (en este proyecto no usa defer en ese tag).

12. Tras acertar, ¿qué le pasa al input?

  1. inputNumero.readOnly = true
  2. inputNumero.disabled = true
  3. Se borra su id.
Respuesta

B. Queda deshabilitado para no seguir escribiendo.

13. “Tibio, tibio” aplica cuando la distancia cumple:

  1. >= 15 y < 50 (por el orden de los if).
  2. Solo si la distancia es exactamente 15.
  3. Si la distancia es menor que 15.
Respuesta

A. Primero se descarta >= 50 (frío); luego >= 15 (tibio); lo que queda es caliente.

14. ¿Qué lanza el código si falta el botón con id="boton" en el HTML?

  1. Un alert al usuario.
  2. Un Error con throw que verás en la consola.
  3. Nada; el juego sigue.
Respuesta

B. El bloque inicial comprueba !boton y hace throw new Error(...).

15. ¿Qué validación usa el JS para rechazar el valor del input?

  1. Cualquier número decimal es válido.
  2. Debe ser entero y estar entre 1 y 499 (inclusive).
  3. Solo comprueba que no esté vacío.
Respuesta

B. Number.isInteger(valor) y límites valor < 1 || valor > 499.
