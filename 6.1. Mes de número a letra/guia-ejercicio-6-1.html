Guía del ejercicio 6.1

Por qué estos cambios en el JavaScript (array de objetos, findIndex, mensaje con días).

1. De textos con null a meses con objetos

Antes había una lista de nombres y un null al inicio para que la posición 1 fuera enero. Ahora cada mes es un objeto con numeroMes, mesLetra y numeroDias. El array se llama meses como pide el enunciado y encaja con meses.findIndex(...).

2. Quitar null

Cada objeto ya trae su numeroMes (1, 2, 3…). No hace falta reservar la posición 0 del array.

3. Variable mesNumero

Es el nombre que pide la práctica para el mes que introduce el usuario (entre 1 y 12). Es el mismo valor que lees del input; solo cambia el nombre para coincidir con el guion.

4. findIndex en lugar de [índice]

Antes el índice del array coincidía con el número del mes. Ahora lo importante es el campo numeroMes dentro de cada objeto. findIndex recorre meses y devuelve la posición del objeto cuyo numeroMes coincide con lo escrito.

5. mesElegido y el mensaje con días

Con el índice, meses[indice] es el mes completo. De ahí salen mesLetra y numeroDias para la frase: número → nombre → días.

6. Usar mesNumero en los if

Si renombraste la variable pero en las comprobaciones seguía el nombre antiguo (numeroMes), esa variable ya no existe y el navegador muestra error. Hay que usar mesNumero en todas las comprobaciones del número del usuario.

En una frase: pasamos de «lista de nombres por posición» a «fichas de mes» en un array, buscamos por numeroMes, mostramos nombre y días, y usamos los nombres de variable del enunciado.
