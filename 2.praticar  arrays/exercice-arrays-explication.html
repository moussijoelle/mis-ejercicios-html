Exercice « Practicar arrays »

Fiche de révision : énoncé, ce qu’on a mis en place dans le projet (index.html + app.js), et 15 questions avec réponses.

1. L’énoncé (consigne)

  • Créer en JavaScript un tableau au choix (citations, animaux, fleurs, etc.).
  • Deux boutons : l’un augmente l’index affiché, l’autre le diminue.
  • À chaque changement valide, afficher l’élément du tableau à cet index dans la page.
  • Pistes : utiliser if pour ne pas descendre sous 0 et ne pas dépasser le dernier index (longueur − 1). La variable d’index doit être déclarée en dehors des fonctions.

2. Ce qu’on a fait dans le projet

HTML (index.html)

  • Texte de l’énoncé visible sur la page (rappel du travail demandé).
  • Une zone en flex : bouton « Disminuir », paragraphe central vide avec id="citaActual", bouton « Aumentar ».
  • Le paragraphe est vide dans le HTML : c’est le JavaScript qui y écrit la citation.
  • type="button" sur les boutons pour éviter un envoi de formulaire accidentel.
  • Chargement du script en fin de body : <script src="app.js"></script> pour que le DOM (notamment #citaActual) existe déjà.

JavaScript (app.js)

  • Tableau citas : objets { texto, autor }.
  • Variable posicionAMostrar (index courant), déclarée au niveau du fichier.
  • document.getElementById("citaActual") pour cibler le paragraphe.
  • Fonction mostrarEnPosicion() : lit citas[posicionAMostrar] et met à jour innerHTML.
  • aumentar() : if tant que l’index est strictement inférieur à citas.length - 1, puis ++ et rafraîchissement.
  • disminuir() : if tant que l’index est > 0, puis -- et rafraîchissement.
  • Appel initial mostrarEnPosicion() pour afficher la première citation au chargement.

Par rapport au HTML de la professeure : pas de bloc <details> ni citation écrite en dur au centre ; à la place, un id et le fichier app.js pour que l’affichage suive l’index.

2 bis. Données concrètes de l’exercice (comment le tableau est appliqué)

Consigne (énoncé du cours, en espagnol) : « Crée un tableau de ce que tu veux (citations, animaux, fleurs) en JS ».

Choix dans ce projet : un tableau de citations (citas), pas d’animaux ni de fleurs. Chaque élément est un objet avec texto (la phrase) et autor (l’auteur ou la source : anonyme, proverbe, etc.).

Code dans app.js (identique au projet) :

const citas = [
  { texto: "Lo único imposible es aquello que no intentas.", autor: "Anónimo" },
  { texto: "El que no arriesga, no gana.", autor: "Refrán" },
  { texto: "Poco a poco se va lejos.", autor: "Proverbio" },
];

Correspondance des indices à l’écran :

  • posicionAMostrar === 0 → première citation (Anónimo).
  • posicionAMostrar === 1 → deuxième (Refrán).
  • posicionAMostrar === 2 → troisième (Proverbio) — dernière position valide.

Utilisation : citas[posicionAMostrar] donne l’objet courant ; mostrarEnPosicion() écrit item.texto et item.autor dans #citaActual. Les boutons ne modifient posicionAMostrar qu’entre 0 et citas.length - 1 (ici 2), car citas.length vaut 3.

3. 15 questions — réponses

Clique sur une question pour afficher la réponse.

1. Qu’est-ce qu’un tableau (array) en JavaScript ?
C’est une liste ordonnée de valeurs dans une seule variable. On accède à chaque valeur avec un index numérique commençant à 0.
2. Pourquoi le premier élément est à l’index 0 ?
C’est une convention du langage : la distance depuis le début du tableau. Le premier élément n’est « à aucune case » avant lui, donc index 0.
3. Que signifie citas[posicionAMostrar] ?
« Prendre dans le tableau citas l’élément à la position posicionAMostrar. » Par exemple, si l’index vaut 1, on obtient le deuxième objet du tableau.
4. Pourquoi posicionAMostrar est en dehors des fonctions ?
Pour qu’un seul compteur soit partagé par aumentar, disminuir et mostrarEnPosicion. Si la variable était à l’intérieur d’une fonction, elle serait réinitialisée ou locale à cette fonction seulement.
5. Pourquoi dans aumentar on compare à citas.length - 1 ?
Dans ce fichier app.js, citas a 3 éléments, donc les index valides sont 0, 1, 2. Le dernier est toujours longueur − 1 (ici 2). On n’augmente que si l’index actuel est strictement plus petit que ce dernier index.
6. Que se passerait-il sans le if dans disminuir ?
On pourrait passer à l’index -1, -2… ce qui ne correspond pas à un élément du tableau : affichage undefined ou erreur selon le code.
7. Que font posicionAMostrar++ et posicionAMostrar-- ?
++ ajoute 1 à la variable ; -- enlève 1. C’est équivalent à posicionAMostrar = posicionAMostrar + 1 (ou - 1).
8. À quoi sert document.getElementById("citaActual") ?
À récupérer l’élément HTML unique qui a l’attribut id="citaActual", pour pouvoir modifier son contenu depuis le JavaScript.
9. Pourquoi utiliser innerHTML pour la citation ?
Parce qu’on insère du HTML (<br>, <em>). Avec textContent, les balises s’afficheraient comme du texte brut au lieu d’être interprétées.
10. Pourquoi le <script> est placé en bas du body ?
Pour que le navigateur ait déjà créé les balises (dont #citaActual) avant d’exécuter app.js. Sinon, getElementById peut retourner null.
11. Pourquoi séparer le JS dans app.js ?
Pour organiser le projet : HTML = structure, JS = logique. Plus facile à relire, corriger et réutiliser qu’un immense bloc <script> dans le HTML.
12. Pourquoi le <p id="citaActual"> est vide dans le HTML ?
Le texte affiché dépend de l’index. On laisse un « conteneur » vide ; le JS remplit ce conteneur selon posicionAMostrar. La professeure avait la première citation en dur pour l’exemple visuel ; ici, on la génère dynamiquement.
13. À quoi sert le flex sur le conteneur des boutons ?
À aligner en ligne les deux boutons et la zone centrale, avec de l’espace entre eux, comme dans l’exemple du cours.
14. Pourquoi type="button" sur les boutons ?
Le type par défaut dans un formulaire est parfois « submit ». Avec type="button", le clic ne fait qu’exécuter onclick sans risquer d’envoyer un formulaire.
15. Pourquoi citas est en const et posicionAMostrar en let ?
const : la variable citas continue à pointer vers le même tableau (on ne réassigne pas citas = ... ailleurs). let pour posicionAMostrar car sa valeur numérique change à chaque clic valide.
