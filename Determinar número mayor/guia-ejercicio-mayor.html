En qué consistía el ejercicio

El ejercicio pedía una página web sencilla donde el usuario escribe dos números enteros, pulsa un botón y la página muestra cuál es el mayor. Si lo escrito no son dos enteros válidos, debe mostrarse un mensaje de error en lugar del resultado. Además se usó HTML, estilos con Tailwind (por CDN), un poco de diseño (colores, bordes, imagen) y un archivo app.js con la lógica en JavaScript.

15 preguntas con respuesta

  1. ¿Para qué sirve document.getElementById?
    Respuesta: Obtiene del HTML el elemento que tiene el id indicado para poder leerlo o modificarlo desde JavaScript.
  2. ¿Por qué se usa parseInt(..., 10) y no solo parseInt(...)?
    Respuesta: El segundo argumento fuerza base decimal y evita sorpresas con ceros a la izquierda en motores antiguos.
  3. ¿Qué significa Number.isNaN(a)?
    Respuesta: Devuelve verdadero si a no es un número (por ejemplo tras un parseInt fallido con NaN).
  4. ¿Qué hace Math.max(a, b) si los dos números son iguales?
    Respuesta: Devuelve ese mismo número (el máximo de dos iguales es el valor común).
  5. ¿Por qué el botón lleva type="button"?
    Respuesta: Un botón dentro de un formulario podría enviarlo; aquí no queremos envío, solo ejecutar JS al clic.
  6. ¿Para qué sirve el atributo for en <label>?
    Respuesta: Asocia la etiqueta al control con el mismo id; mejora usabilidad y accesibilidad.
  7. ¿Qué diferencia hay entre textContent e innerHTML para el mensaje?
    Respuesta: textContent trata el texto como texto plano (más seguro); innerHTML interpretaría etiquetas HTML.
  8. ¿Por qué el <script src="app.js"> va al final del body?
    Respuesta: Así el navegador ya ha creado los elementos del DOM antes de ejecutar el código que los busca con getElementById.
  9. ¿Qué hace addEventListener("click", calcularMayor)?
    Respuesta: Cuando el usuario hace clic en el botón, el navegador ejecuta la función calcularMayor.
  10. ¿Para qué es inputmode="numeric"?
    Respuesta: Pista al dispositivo para mostrar teclado numérico en móviles (no impide escribir letras en escritorio).
  11. ¿Qué papel cumple id="resultado"?
    Respuesta: Identifica el elemento donde el script escribe el mensaje de error o de éxito.
  12. ¿Qué indica lang="es" en <html>?
    Respuesta: El idioma principal del documento es español (lectores de pantalla, corrección ortográfica del navegador, SEO).
  13. ¿Qué logra class="max-w-32" en el input?
    Respuesta: Limita el ancho máximo del campo para que no se estire demasiado en horizontal.
  14. ¿Qué hace return dentro de calcularMayor tras el error?
    Respuesta: Termina la función en ese punto para no ejecutar la línea que muestra el mayor con datos inválidos.
  15. ¿Qué es el <!DOCTYPE html>?
    Respuesta: Declaración que dice que el documento es HTML5 y ayuda al navegador a renderizar en modo estándar.
