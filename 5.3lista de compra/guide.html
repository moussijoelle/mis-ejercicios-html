Guide – Exercice “Liste de courses”

Ce document reprend tes dernières questions + réponses, pour suivre l’exercice étape par étape.

Resumen del ejercicio (en español)

Vas a construir una pequeña lista de compra (carrito) con 3 acciones: agregar(), eliminarPorPosicion() y vaciarCarrito().

  • Guardar los productos en un array llamado carrito.
  • Mostrar el array en el HTML (lista) y mostrar cuántos productos hay (contador).
  • Eliminar por posición usando slice() (sin splice()).
  • Vaciar el carrito usando carrito = [].
  • Tener la lógica en app.js y enlazarlo desde el HTML.

JavaScript (logique de l’exercice)

Q: “Pourquoi on commence par let carrito = [] ?”

R: Parce que tu as besoin d’un endroit pour stocker les produits. On utilise let (pas const) car l’énoncé veut vider avec carrito = [] (réassignation).

Q: “array = [] à quoi ça sert ? Qu’est-ce que la prof veut qu’on apprenne ?”

R: Ça sert à vider la liste d’un coup : tu jettes l’ancienne liste et tu en crées une nouvelle vide. Tu apprends la notion de “réinitialiser l’état” et la différence entre réassigner (= []) et modifier une liste.

Q: “let carrito = [] doit être la première ligne ?”

R: Pas obligatoire, mais ça doit être défini avant les fonctions qui l’utilisent (ajouter, supprimer, vider, render). Le plus simple est de le mettre tout en haut.

Q: “render ça signifie quoi ?”

R: “Render” = “dessiner/afficher”. render() efface la liste à l’écran, puis réécrit tous les produits du tableau carrito dans le HTML, et met à jour le compteur.

Q: “Le prof dit d’utiliser slice, ça ne va pas gêner ?”

R: slice() ne supprime pas “dans” le tableau, il copie. Pour supprimer avec slice, tu reconstruis un nouveau tableau : “avant la position” + “après la position”. 

carrito = carrito.slice(0, pos).concat(carrito.slice(pos + 1));

Q: “Explique slice vs splice comme à une enfant de 5 ans.”

R: Imagine que carrito est une rangée de jouets. slice = tu fais une photo de quelques jouets (copie) sans toucher la rangée. splice = tu prends ta main et tu enlèves (ou ajoutes) un jouet dans la rangée (ça change la vraie liste). 

// slice: copie (ne modifie pas)
const photo = carrito.slice(0, 2);

// splice: modifie (supprime à la position pos)
carrito.splice(pos, 1);

Mini vocabulaire (boutique virtuelle)

Q: “C’est quoi une étiquette ?”

R: Une étiquette = un nom de variable. Tu donnes un nom à quelque chose pour le retrouver (ex: carrito, inputProducto).

Q: “Explique-moi let / const / array / objet / attribut / function…”

  • let: étiquette qu’on peut changer
  • const: étiquette fixe (on ne remplace pas la référence)
  • array ([]): liste
  • objet ({}): fiche avec des infos
  • attribut (HTML): info dans une balise, ex id="..."
  • propriété (JS): info d’un objet, ex innerHTML
  • méthode: action d’un objet, ex insertAdjacentHTML()
  • function: recette/action à appeler, ex render()

CSS — en qué consiste en tu proyecto

En tu HTML usas Tailwind por CDN y además tienes una capa de estilos base dentro de <style type="text/tailwindcss">.

  • Tailwind (CDN): te permite usar clases como flex, bg-blue-500, w-5/6, etc.
  • @apply: “mezcla” utilidades de Tailwind dentro de reglas CSS (por ejemplo para button, input).
  • Objetivo: que los botones e inputs tengan estilo sin repetir muchas clases en cada etiqueta.
