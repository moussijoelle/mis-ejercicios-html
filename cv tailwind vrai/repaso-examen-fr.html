Ce texte s’adresse à quelqu’un qui ne connaît pas la programmation : on parle comme d’une page dans un magazine ou d’un document Word, mais affiché dans le navigateur. Les textes de ton CV restent en espagnol ; ici on explique seulement les couleurs, la disposition et les photos d’exemple.

Dans l’exemple ci-dessous (fond sombre, comme dans beaucoup d’éditeurs), chaque couleur aide à repérer le rôle d’un morceau de texte.

</header>

Quatre images mentales (sans jargon)

  1. La page web, c’est comme une feuille : il y a des titres, des liens, des tableaux. En informatique on appelle ça du « HTML », mais tu peux penser : le contenu.
  2. Le style, c’est la déco : couleurs, tailles, alignement. Avant, une grande partie était dans un fichier à part (souvent .css), comme une feuille de consignes pour le graphiste.
  3. Tailwind, c’est une façon de noter la déco à côté de chaque morceau de texte, avec de petits mots dans class="…" — comme des étiquettes collées sur chaque bloc.
  4. La ligne « CDN », c’est une phrase toute prête en haut du fichier qui dit au navigateur : « va chercher Tailwind sur Internet ». Tu n’as pas besoin d’installer un gros programme sur ton PC pour ce projet.

  1. « Mon CV, ce sont plusieurs fichiers que j’ouvre dans le navigateur, comme des pages d’un site. »
  2. « Pour la mise en forme j’utilise Tailwind : c’est un outil sur Internet ; j’ai ajouté une ligne toute faite en haut de chaque fichier. »
  3. « Les couleurs du menu violet, le texte blanc, etc., ce sont des mots-clés à côté du texte — un peu comme des styles dans Word, mais écrits à la main dans le fichier. »

Rappel — une phrase avant, une phrase après

Avant : la plupart des couleurs et alignements étaient écrits dans un fichier séparé (souvent .css), un peu loin du texte du CV.

Après : une ligne magique en haut du fichier active Tailwind ; ensuite, à côté de chaque bloc de page, on met des petits mots dans class="…" pour dire « violet », « centré », « marge en haut », etc.

Le contenu (ce que lit le recruteur) ne change pas de langue ; change surtout la manière de décrire la mise en page.

Les « classes » — sans faire peur

Une classe, ici, ce n’est pas l’école : c’est juste un mot-clé dans class="…". Plusieurs mots = plusieurs idées (couleur + centrage + taille), séparées par des espaces.

  • bg-… = couleur de fond. Exemple : bg-[#58008b] = fond violet (le code couleur est entre crochets).
  • text-… = couleur ou taille du texte. Exemple : text-white = texte blanc.
  • flex + justify-center = mettre des éléments en ligne et les centrer (souvent le menu).
  • mx-auto = centrer un bloc (par exemple la photo).
  • hover:underline = quand on passe la souris sur un lien, il se souligne.
  • md:text-3xl = sur un grand écran, le texte devient plus gros (le préfixe md: = « à partir de cette largeur »).

À l’oral : montre la photo de l’écran avec le doigt : « ici c’est violet, ici c’est centré » — puis : « c’est réglé avec Tailwind ». Personne ne te demande de réciter l’alphabet des mots techniques.

Plus tard : détails pour niveau avancé

On peut aussi utiliser grid, gap-, border-, des valeurs entre crochets pour ombres ou marges précises, etc. Les étapes ci-dessous gardent des listes plus techniques si tu veux approfondir.

Comment chercher du « code » Tailwind (bases pour débutant)

Tu n’inventes pas les mots au hasard : tu vas sur la documentation officielle, comme un dictionnaire. Tout est gratuit et en ligne.

  1. Où aller : ouvre le site de la doc Tailwind (en anglais, mais très visuel) : https://tailwindcss.com/docs. C’est la référence pour le CDN que tu utilises.
  2. Comment t’y retrouver : à gauche (ou dans le menu « hamburger » sur téléphone), il y a des rubriques : par exemple Layout (mise en page), Flexbox & Grid, Spacing (marges et espacements), Sizing (largeur/hauteur), Typography (texte), Backgrounds (fonds), Borders, Effects (ombres…). Tu cliques sur ce qui ressemble à ce que tu veux faire.
  3. Chercher un mot précis : sur la page de la doc, utilise Ctrl+F (Windows) ou Cmd+F (Mac) et tape le début du mot (flex, center, padding…). Tu tombes souvent sur un tableau avec la classe à copier et un aperçu.
  4. Lire une ligne du tableau : à gauche le nom de la classe Tailwind (ex. text-center), à droite l’équivalent en CSS classique. Tu copies le nom de gauche dans ton class="…".
  5. Couleur exacte (comme ton violet) : si la palette proposée ne suffit pas, la doc explique les valeurs entre crochets : bg-[#58008b] = « fond de cette couleur précise ». Cherche la section sur les couleurs ou « arbitrary values ».
  6. Tester sans casser ton CV : tu peux ouvrir la démo en ligne (playground) sur le même site : tu tapes du HTML avec des classes et tu vois le résultat tout de suite — pratique pour essayer avant de coller dans ton fichier.

Astuce : dans ton propre projet, cherche d’abord une page qui ressemble à ce que tu veux (ex. le menu de index.html), copie les classes qui marchent, puis va dans la doc pour comprendre mot par mot ce qu’elles font.

Étape 0 — Par où on commence

Tu avais déjà ton CV sur plusieurs pages (plusieurs fichiers que tu ouvres dans le navigateur). La « décoration » était surtout dans un autre fichier, à côté.

Le but : que le site ressemble pareil pour les visiteurs, mais que toi tu expliques plus facilement où tu as mis les couleurs et les alignements — en grande partie à côté du texte, avec Tailwind.

Ce qu’on a utilisé pour coder (à cette étape)

  • Des fichiers HTML (une page = un fichier) et, avant Tailwind, souvent un fichier CSS séparé (ex. estilos.css) relié avec <link rel="stylesheet" …>.
  • Un navigateur (Chrome, Edge…) pour voir le résultat après chaque modification.

Comment reproduire la même situation de départ

  1. Crée un petit dossier avec deux fichiers : page.html + style.css.
  2. Dans le HTML, mets le texte du CV dans des balises simples (h1, p) et, dans le head, le link vers style.css.
  3. Dans le CSS, mets les couleurs et marges : tu verras que la « déco » est loin du texte dans le fichier — c’est exactement ce qu’on voulait améliorer avec Tailwind à l’étape suivante.

Étape 1 — Activer Tailwind (sur chaque page)

En haut de chaque fichier de page, il y a une zone « technique » appelée <head> — pense : l’en-tête que le navigateur lit en premier, avant d’afficher la page.

  • On enlève (ou on n’utilise plus) le lien vers l’ancien fichier de style estilos.css si c’était lui qui faisait tout.
  • On colle une ligne qui charge Tailwind depuis Internet : <script src="https://cdn.tailwindcss.com"></script>
  • On garde les lignes pour les accents et le mobile (charset, viewport) — ce sont des réglages de base.

Le navigateur (Chrome, Edge…) télécharge Tailwind tout seul. Tu n’as pas besoin d’un « gros logiciel de développement » installé pour que ton CV s’affiche.

Ce qu’on a utilisé pour coder

  • La play CDN officielle : une balise <script src="https://cdn.tailwindcss.com"></script> dans le head de chaque page du CV.
  • Les balises HTML de base : meta charset, meta viewport, title.
  • Un éditeur de texte (VS Code, Cursor…) pour coller la ligne et enregistrer.

Comment reproduire pareil sur une nouvelle page

  1. Ouvre ton fichier .html.
  2. Juste après <meta name="viewport" …> (ou avant <title>), colle exactement : <script src="https://cdn.tailwindcss.com"></script>.
  3. Supprime ou commente la ligne <link rel="stylesheet" href="estilos.css"> si tu ne veux plus du vieux fichier CSS.
  4. Enregistre, ouvre la page dans le navigateur, appuie sur F5 : Tailwind est actif ; tu peux maintenant écrire des classes dans class="…".

Étape 2 — index.html (accueil)

Cette page, c’est quoi ? La page d’accueil : le menu pour aller partout, ta photo, ton nom, le texte du profil, et en bas ton mail et tes liens.

Ce qu’on a utilisé pour coder

  • HTML : header, nav, a, main, img, footer.
  • Tailwind uniquement via l’attribut class="…" (pas de règles maison dans un .css pour cette page).
  • Des utilitaires typiques : flex, justify-center, bg-[#58008b], text-white, mx-auto, hover:underline, petites icônes svg avec h-4 w-4.

Comment reproduire pareil

  2. Pour la photo : garde le chemin src="…" correct ; sur la balise img, ajoute class="mx-auto … max-w-full" comme dans le fichier.
  3. Va sur tailwindcss.com → Flexbox et Backgrounds si tu veux changer une couleur : cherche le mot-clé, copie l’exemple dans ton class.
  4. À chaque essai : Ctrl+S puis F5 dans le navigateur.

Étape 3 — educación.html (formation)

Cette page, c’est quoi ? Le tableau des études (niveau, mention, centre, année), avec un petit lien « maison » pour revenir à l’accueil.

Sur la capture (Figure 2) : un grand tableau avec cadre doré, ligne d’en-tête violette, cases colorées (lavande ou jaune). La colonne du milieu regroupe trois mentions dans des petits blocs blancs.

En simple : Tailwind sert surtout aux bordures, aux fonds (violet, lavande, jaune) et au texte blanc dans l’en-tête. La fusion de cellules (rowspan="3") est du HTML de tableau, pas une classe Tailwind.

Ce qu’on a utilisé pour coder

Comment reproduire pareil

  1. Dessine d’abord ton tableau sur papier ou dans un tableur (colonnes / lignes).
  2. Recopie la structure depuis educación.html : mêmes balises, puis remplace le texte par tes études.
  3. Colle les mêmes classes sur table, th et td ; ajuste les couleurs entre crochets bg-[#…] si tu veux une autre charte.
  4. Pour une fusion de lignes : garde le rowspan="…" en HTML (la doc Tailwind ne remplace pas ça).
Détail des classes (niveau avancé)
  • Table : border-4 border-[rgb(139,125,0)], w-full border-collapse
  • En-têtes : bg-[#58008b] text-white
  • Cellules : bg-[#f3e9ff] ou bg-[#e8d45a]
  • Bloc mentions : flex flex-col gap-2 + petits bg-white

Étape 4 — experiencia.html

Cette page, c’est quoi ? La liste de tes expériences pro, présentée comme des accordéons : tu cliques sur le titre violet pour ouvrir ou fermer le texte.

Sans jargon : chaque emploi est un bloc repliable. En HTML ça s’appelle details / summary, mais tu peux dire : « comme une FAQ où on déplie la réponse ». Le premier bloc est déjà ouvert au chargement.

Sur la capture (Figure 3) : fond jaune par bloc ; bandeau violet cliquable avec le titre du poste ; texte en dessous quand c’est ouvert.

En simple : classes pour le fond jaune du bloc, le fond violet du titre, le texte blanc sur le titre, et un peu d’espace entre les blocs.

Ce qu’on a utilisé pour coder

  • HTML natif : <details> (bloc repliable) et <summary> (ligne cliquable) — fournis par le navigateur, pas besoin de JavaScript.
  • Attribut open sur le premier details pour qu’il soit déplié au chargement.
  • Tailwind sur details et summary + sur le paragraphe du contenu (marges, centrage).

Comment reproduire pareil

  1. Copie un bloc complet depuis experiencia.html : <details>…</details>.
  2. Change le texte du summary (nom du poste) et le paragraphe en dessous.
  3. Duplique le bloc pour un nouvel emploi ; enlève open des copies si tu veux qu’elles restent fermées au départ.
  4. Garde les mêmes class="…" sur details et summary pour garder le même style accordéon.
Détail des classes
  • details : mb-8 bg-[#e4cf15] py-[10px]
  • summary : bg-[#58008b] text-white text-[1.2rem]
  • Texte : m-[3px] text-center

Étape 5 — certificados.html

Cette page, c’est quoi ? Tes diplômes et certificats : un titre, un paragraphe d’explication en dessous, parfois un lien vers le site de la formation.

Côté technique (facultatif à dire) : une structure « titre + texte » en HTML (dl / dt / dd). Pour un non-programmeur : comme une liste à deux niveaux dans Word.

Sur la capture (Figure 4) : titres en violet, texte en dessous avec une barre violette à gauche pour bien séparer chaque bloc.

En simple : classes pour le gras, la couleur violette, le soulignement au survol sur les liens, et une bordure gauche fine sur les paragraphes.

Ce qu’on a utilisé pour coder

  • HTML liste définitions : dl > dt (titre du certificat) + dd (description / lien).
  • Tailwind : font-bold, text-[#58008b], underline, hover:text-[#3d0059], border-l-[3px], pl-4, max-w-[65ch].

Comment reproduire pareil

  1. Pour chaque certificat, ajoute une paire <dt>…</dt> puis <dd>…</dd> dans le dl.
  2. Copie les class du dt et du dd depuis certificados.html.
  3. Si tu ajoutes un lien dans le dt, garde les classes sur la balise a pour le survol violet.
Détail des classes
  • dt : font-bold text-[#58008b], liens avec underline hover:text-[#3d0059]
  • dd : max-w-[65ch] border-l-[3px] pl-4

Étape 6 — Idiomas.html

Cette page, c’est quoi ? Une carte du monde : tu cliques sur une région pour descendre vers le texte sur ce pays.

Idée simple : une grande image + des zones cliquables par-dessus (comme un plan interactif). Les noms techniques importent peu à l’oral : « carte cliquable » suffit.

Sur la capture (Figure 5B) : gros titre, carte du monde qui s’adapte à la largeur de l’écran, textes en dessous.

En simple : le titre peut devenir plus gros sur grand écran (md:text-3xl). L’image de la carte est large mais limitée pour rester lisible (w-full max-w-[1100px]).

Ce qu’on a utilisé pour coder

  • Un fichier image img/world.svg + une image map HTML (map, area) pour les clics — la logique de la carte reste du HTML classique.
  • Tailwind surtout sur le h1, l’icône et le conteneur de l’image : flex, md:text-3xl, mx-auto w-full max-w-[1100px].

Comment reproduire pareil

  1. Ne touche pas aux coordonnées de la carte si tout fonctionne : concentre-toi sur les class autour.
  2. Compare avec Idiomas.html : copie le bloc titre + classes, puis le bloc img avec les mêmes classes.
  3. Pour le responsive du titre, cherche « Responsive design » sur tailwindcss.com et teste en réduisant la fenêtre du navigateur.
Détail des classes
  • h1 : flex flex-wrap justify-center md:text-3xl
  • Icône : md:h-10 md:w-10
  • Image carte : mx-auto w-full max-w-[1100px]

Étape 7 — tecnologia.html

Cette page, c’est quoi ? La liste de tes compétences techniques (langages, outils, bases de données…).

Sur la capture (Figure 6) : une boîte jaune au centre avec une liste à puces et des sous-listes numérotées ; une ombre autour pour la faire ressortir.

En simple : le bloc est centré, pas toute la largeur de la page ; fond jaune un peu transparent ; coins arrondis ; ombre violette légère.

Ce qu’on a utilisé pour coder

  • HTML : ul / ol / li pour la liste des techno.
  • Tailwind : un div (ou section) parent avec mx-auto w-fit pour centrer une boîte « à la taille du contenu », plus rounded, shadow-[…], couleur de fond en hex avec transparence bg-[#e6bd07de].

Comment reproduire pareil

  1. Enveloppe toute ta liste dans un <div class="mx-auto w-fit …"> copié depuis tecnologia.html.
  2. Ajoute ou enlève des <li> : le cadre jaune s’adapte tout seul.
  3. Pour une autre couleur de fond, change les 8 caractères hex dans bg-[#……] (les 2 derniers = transparence en notation hex).
Détail des classes
  • mx-auto w-fit = centrer une boîte à la taille du texte
  • bg-[#e6bd07de] = jaune avec transparence (les 2 derniers caractères)
  • shadow-[…] = ombre personnalisée
  • Sous-listes : list-decimal space-y-0.5

Étape 8 — softskills.html

Cette page, c’est quoi ? Un titre et une vidéo de présentation (comme sur YouTube, mais sur ton CV).

Pourquoi des réglages sur la vidéo ? Pour que ça marche bien sur téléphone (lecture dans la page, pas plein écran bizarre). Tu peux dire simplement : « vidéo intégrée avec les boutons lecture/pause ».

Sur la capture (Figure 7) : titre + vidéo centrée, pas trop large sur grand écran.

En simple : classes pour centrer la vidéo et limiter sa largeur (max-w-[600px]).

Ce qu’on a utilisé pour coder

  • La balise HTML <video> avec src, controls, playsinline (bon comportement sur mobile).
  • Tailwind sur la vidéo : mx-auto, block, max-w-[600px], w-full, etc.

Comment reproduire pareil

  1. Mets ton fichier vidéo (ex. .mp4) dans le dossier du projet (souvent img/) et vérifie le chemin dans src="…".
  2. Copie la balise video complète depuis softskills.html avec les mêmes class.
  3. Teste sur téléphone ou mode responsive du navigateur : garde playsinline si la lecture doit rester dans la page.

Étape 9 — aficiones.html

Cette page, c’est quoi ? Tes loisirs : texte, photos (chat, GIFs), parfois un petit tableau ludique. Même idée que les autres pages : bandeau violet en haut, contenu au milieu.

Sur les trois figures (8a, 8b, 8c) : ce sont des vues différentes de la même page (haut, milieu, bas). Tu peux dire la même chose : menu violet, titre, images, tableau sur les chats.

En simple : comme les autres pages : fond violet dans l’en-tête, tableau avec bordures violettes, cases lavande. S’il y a une petite grille de photos en haut, c’est une grille (grid) qui passe de 1 à 3 colonnes selon l’écran.

Ce qu’on a utilisé pour coder

  • HTML : texte, img, table, form, input, textarea, button, éventuellement details pour montrer du code.
  • Tailwind : grid + préfixes sm: / lg: pour la galerie ; mêmes idées border / bg-* que sur les autres pages ; on a retiré les anciennes classes inventées (sans Tailwind) pour tout écrire en utilitaires.

Comment reproduire pareil

  1. Ouvre aficiones.html et repère section par section (titre, images, tableau, formulaire).
  2. Pour chaque balise, vérifie qu’il n’y a pas de « faux nom » de classe tout seul : chaque mot dans class doit être un utilitaire Tailwind (ou presque) — compare avec la doc ou une autre page du CV déjà finie.
  3. Pour la grille de cartes : copie le conteneur class="grid … sm:grid-cols-2 lg:grid-cols-3" et change seulement le texte / les images à l’intérieur.
  4. Doc utile : Grid template columns sur le site officiel.
Détail des classes
  • h1 en flex centré
  • Galerie : grid sm:grid-cols-2 lg:grid-cols-3, cartes bg-[#f3e9ff] border-[#58008b]/30
  • Tableau : border border-[#58008b], en-têtes bg-[#58008b] text-white

  3. Dans le tableau des études, en langage simple : que fait rowspan="3" sur une cellule ?
  4. Cite deux classes Tailwind utilisées sur le menu de l’accueil pour aligner les liens en ligne et les centrer (réponse d’exemple acceptée).
  5. Où écrit-on les classes Tailwind dans le fichier : dans quel attribut HTML ?
  6. En une phrase : à quoi sert le préfixe bg- dans une classe (ex. bg-white) ?
  7. Selon le tableau des captures de ce document : quel nom de fichier pour la photo de la page certificados ?

  9. Pourquoi faut-il ouvrir repaso-examen-fr.html dans le bon dossier (à côté de img) pour voir les images ?
  10. À quoi sert le préfixe hover: devant une classe (ex. hover:underline) ?

Réponses courtes (1 à 10)

  1. <script src="https://cdn.tailwindcss.com"></script>
  2. details = le bloc entier qu’on ouvre/ferme ; summary = la ligne visible (souvent violette) sur laquelle on clique.
  3. La cellule s’étend sur trois lignes du tableau (fusion verticale).
  4. Exemples : flex + justify-center (ou aussi flex-wrap, items-center).
  5. Dans l’attribut class="…" sur la balise concernée.
  6. Ça concerne la couleur de fond (background) de l’élément.
  7. images-04.png (dans img/repaso/).
  8. https://tailwindcss.com/docs
  9. Parce que les chemins vers les images sont relatifs : si le fichier HTML n’est pas au bon endroit, le navigateur ne trouve pas le dossier img/.
  10. Le style s’applique quand on passe la souris sur l’élément (survol).

Diagnostic — pourquoi je ne vois pas les photos

Ici, le site essaie d’ouvrir chaque image comme le ferait un lecteur. Si tout est « OK », les fichiers sont au bon endroit avec le bon nom. Si tu vois « NON », c’est comme une pièce jointe manquante dans un mail : le fichier n’est pas là ou le nom ne correspond pas.

Ouvre repaso-examen-fr.html par double-clic dans le dossier du projet (avec le dossier img à côté). Si tu copies seulement ce fichier sur le Bureau sans le dossier des images, ça ne peut pas marcher.

Si presque tout est NON : vérifie le dossier img/repaso/ et les noms images-01 … images-08 (fichier .png, .jpg ou .jpeg), plus pour Aficiones images-08-b et images-08-c si tu les utilises.

Astuce : la carte du monde sur la page des langues devrait toujours apparaître. Si elle est absente, tu n’es pas dans le bon dossier.
