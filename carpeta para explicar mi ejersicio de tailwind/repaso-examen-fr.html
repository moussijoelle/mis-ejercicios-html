Quatre images mentales (sans jargon)

  1. La page web, c’est comme une feuille : il y a des titres, des liens, des tableaux. En informatique on appelle ça du « HTML », mais tu peux penser : le contenu.
  2. Le style, c’est la déco : couleurs, tailles, alignement. Avant, une grande partie était dans un fichier à part (souvent .css), comme une feuille de consignes pour le graphiste.
  3. Tailwind, c’est une façon de noter la déco à côté de chaque morceau de texte, avec de petits mots dans class="…" — comme des étiquettes collées sur chaque bloc.
  4. La ligne « CDN », c’est une phrase toute prête en haut du fichier qui dit au navigateur : « va chercher Tailwind sur Internet ». Tu n’as pas besoin d’installer un gros programme sur ton PC pour ce projet.

Rappel — une phrase avant, une phrase après

Après : une ligne magique en haut du fichier active Tailwind ; ensuite, à côté de chaque bloc de page, on met des petits mots dans class="…" pour dire « violet », « centré », « marge en haut », etc.

Le contenu (ce que lit le recruteur) ne change pas de langue ; change surtout la manière de décrire la mise en page.

Les « classes » — sans faire peur

Une classe, ici, ce n’est pas l’école : c’est juste un mot-clé dans class="…". Plusieurs mots = plusieurs idées (couleur + centrage + taille), séparées par des espaces.

  • bg-… = couleur de fond. Exemple : bg-[#58008b] = fond violet (le code couleur est entre crochets).
  • text-… = couleur ou taille du texte. Exemple : text-white = texte blanc.
  • flex + justify-center = mettre des éléments en ligne et les centrer (souvent le menu).
  • mx-auto = centrer un bloc (par exemple la photo).
  • hover:underline = quand on passe la souris sur un lien, il se souligne.
  • md:text-3xl = sur un grand écran, le texte devient plus gros (le préfixe md: = « à partir de cette largeur »).

À l’oral : montre la photo de l’écran avec le doigt : « ici c’est violet, ici c’est centré » — puis : « c’est réglé avec Tailwind ». Personne ne te demande de réciter l’alphabet des mots techniques.

Comment chercher du « code » Tailwind (bases pour débutant)

Tu n’inventes pas les mots au hasard : tu vas sur la documentation officielle, comme un dictionnaire. Tout est gratuit et en ligne.

  1. Où aller : ouvre le site de la doc Tailwind (en anglais, mais très visuel) : https://tailwindcss.com/docs. C’est la référence pour le CDN que tu utilises.
  2. Comment t’y retrouver : à gauche (ou dans le menu « hamburger » sur téléphone), il y a des rubriques : par exemple Layout (mise en page), Flexbox & Grid, Spacing (marges et espacements), Sizing (largeur/hauteur), Typography (texte), Backgrounds (fonds), Borders, Effects (ombres…). Tu cliques sur ce qui ressemble à ce que tu veux faire.
  3. Chercher un mot précis : sur la page de la doc, utilise Ctrl+F (Windows) ou Cmd+F (Mac) et tape le début du mot (flex, center, padding…). Tu tombes souvent sur un tableau avec la classe à copier et un aperçu.
  4. Lire une ligne du tableau : à gauche le nom de la classe Tailwind (ex. text-center), à droite l’équivalent en CSS classique. Tu copies le nom de gauche dans ton class="…".
  5. Couleur exacte (comme ton violet) : si la palette proposée ne suffit pas, la doc explique les valeurs entre crochets : bg-[#58008b] = « fond de cette couleur précise ». Cherche la section sur les couleurs ou « arbitrary values ».
  6. Tester sans casser ton CV : tu peux ouvrir la démo en ligne (playground) sur le même site : tu tapes du HTML avec des classes et tu vois le résultat tout de suite — pratique pour essayer avant de coller dans ton fichier.

Astuce : dans ton propre projet, cherche d’abord une page qui ressemble à ce que tu veux (ex. le menu de index.html), copie les classes qui marchent, puis va dans la doc pour comprendre mot par mot ce qu’elles font.

Guía rápida — cómo hacer cualquier página con Tailwind (CDN)

Esto es la “receta” para crear una página nueva con Tailwind, aunque no sea tu CV. La idea es siempre la misma: HTML = estructura, Tailwind = estilo con clases dentro de class="…".

1) Plantilla mínima (copiar/pegar)

nueva-pagina.html 
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <script src="https://cdn.tailwindcss.com"></script>
  <title>Mi página</title>
</head>
<body class="mx-auto max-w-5xl bg-white px-4 py-6 text-neutral-900">

  <header class="rounded-lg bg-[#58008b] px-3 py-2 text-white">
    <nav class="flex flex-wrap items-center justify-center gap-3 text-sm">
      <a class="no-underline hover:underline" href="index.html">Inicio</a>
      <a class="no-underline hover:underline" href="otra.html">Otra</a>
    </nav>
  </header>

  <main class="mt-6">
    <h1 class="text-2xl font-bold">Título</h1>
    <p class="mt-3 leading-relaxed text-neutral-700">Texto…</p>
  </main>

  <footer class="mt-8 text-center text-xs text-neutral-600">
    &copy; 2026
  </footer>

</body>
</html>

2) Checklist rápida (lo que casi siempre haces)

  • Head: charset + viewport + CDN de Tailwind.
  • Body: ancho cómodo (max-w-… o w-[…]), padding, color base.
  • Layout: elige flex o grid según lo que quieras.
  • Espaciado: usa mt-, mb-, px-, py-, gap-.
  • Responsive: añade sm:, md:, lg: donde haga falta.
  • Probar: Ctrl+S y luego F5 en el navegador.

3) “Diccionario” de clases que más se usan

Layout / alineación

  • flex, flex-wrap, items-center, justify-center
  • grid, sm:grid-cols-2, lg:grid-cols-3

Texto / color / borde

  • text-xs, text-xl, font-bold, leading-relaxed
  • text-white, bg-[#58008b], border, rounded-lg
  • hover:underline, hover:text-[#3d0059]
Plantilla “se parece a tu CV” (misma estética)

Si quieres que una página nueva “parezca” parte del CV (mismo violeta y mismo ancho), copia el header y el body de index.html y cambia solo el contenido del main.

Errores típicos (y solución rápida)

Si algo “no sale”, casi siempre es uno de estos. Mira la línea roja: qué pasa → qué hacer.

  • No se aplica Tailwind → revisa que esté la línea CDN en el <head> y luego Ctrl+S + F5.
  • Las imágenes no aparecen → abre el HTML en la carpeta correcta (rutas relativas) y revisa el nombre exacto del archivo en src="…".
  • Un link no funciona → revisa mayúsculas/minúsculas del archivo (ej. Idiomas.html) y el href.
  • No se centra con mx-auto → recuerda: centra un bloque con ancho. Si hace falta, añade w-… o max-w-….
  • Desborda en móvil → usa max-w-…, w-full y en tablas/código overflow-x-auto.
  • Se ve todo pegado → añade espaciado: mt-, mb-, px-, py-, gap-.
  • El responsive no cambia → revisa prefijos (sm:, md:, lg:) y prueba estrechando la ventana.
  • Copié clases y “no existe” → puede ser un typo: en Tailwind una letra mal y no aplica nada. Copia desde la doc o desde otra página que funciona.
  • SVG enorme en el código → no pasa nada: es normal. Si molesta para estudiar, concéntrate en el class="…" de alrededor.

Glosario rápido (palabras que vas a ver siempre)

Etiqueta (tag)
Una “pieza” de HTML: <header>, <img>, <a>…
Atributo
Información dentro de una etiqueta: class, href, src, alt…
Clase
Palabra(s) en class="…". En Tailwind, cada palabra es un “mini estilo”.
Layout
Cómo se colocan los elementos: flex, grid, centrado, columnas, etc.
Responsive
Que se adapte a móvil/PC. En Tailwind: sm:, md:, lg:.
DOM
El HTML “vivo” dentro del navegador. JavaScript lo lee y lo cambia.
Evento
Algo que pasa: click, escribir, submit… En JS: addEventListener.

Feuille de route — tous les pas du codage avec Tailwind CSS

A. Même routine sur chaque fichier .html

  1. Ouvrir la page dans l’éditeur (Visual Studio Code, Cursor, etc.).
  2. Dans le <head> : ajouter ou vérifier <script src="https://cdn.tailwindcss.com"></script> ; retirer (ou ne plus utiliser) le <link rel="stylesheet" … estilos.css> si c’était l’ancienne méthode.
  3. Garder charset et viewport pour accents et mobile.
  4. Mettre les classes Tailwind sur body (largeur du contenu, police, couleur du texte par défaut).
  5. Mettre les classes sur le header (bandeau violet, texte blanc, lien « maison » ou menu).
  6. Parcourir le main : titres, paragraphes, tableaux, images, formulaires… et remplacer le style « ancien » par des utilitaires Tailwind.
  7. Ouvrir la page dans le navigateur, corriger ce qui cloche (espacements, couleurs), répéter pour la page suivante.

Étape 0 — Par où on commence

Tu avais déjà ton CV sur plusieurs pages (plusieurs fichiers que tu ouvres dans le navigateur). La « décoration » était surtout dans un autre fichier, à côté.

Le but : que le site ressemble pareil pour les visiteurs, mais que toi tu expliques plus facilement où tu as mis les couleurs et les alignements — en grande partie à côté du texte, avec Tailwind.

Ce qu’on a utilisé pour coder (à cette étape)

  • Des fichiers HTML (une page = un fichier) et, avant Tailwind, souvent un fichier CSS séparé (ex. estilos.css) relié avec <link rel="stylesheet" …>.
  • Un navigateur (Chrome, Edge…) pour voir le résultat après chaque modification.

Comment reproduire la même situation de départ

  1. Crée un petit dossier avec deux fichiers : page.html + style.css.
  2. Dans le HTML, mets le texte du CV dans des balises simples (h1, p) et, dans le head, le link vers style.css.
  3. Dans le CSS, mets les couleurs et marges : tu verras que la « déco » est loin du texte dans le fichier — c’est exactement ce qu’on voulait améliorer avec Tailwind à l’étape suivante.

Étape 1 — Activer Tailwind (sur chaque page)

En haut de chaque fichier de page, il y a une zone « technique » appelée <head> — pense : l’en-tête que le navigateur lit en premier, avant d’afficher la page.

  • On enlève (ou on n’utilise plus) le lien vers l’ancien fichier de style estilos.css si c’était lui qui faisait tout.
  • On colle une ligne qui charge Tailwind depuis Internet : <script src="https://cdn.tailwindcss.com"></script>
  • On garde les lignes pour les accents et le mobile (charset, viewport) — ce sont des réglages de base.

Le navigateur (Chrome, Edge…) télécharge Tailwind tout seul. Tu n’as pas besoin d’un « gros logiciel de développement » installé pour que ton CV s’affiche.

Ce qu’on a utilisé pour coder

  • La play CDN officielle : une balise <script src="https://cdn.tailwindcss.com"></script> dans le head de chaque page du CV.
  • Les balises HTML de base : meta charset, meta viewport, title.
  • Un éditeur de texte (VS Code, Cursor…) pour coller la ligne et enregistrer.

Comment reproduire pareil sur une nouvelle page

  1. Ouvre ton fichier .html.
  2. Juste après <meta name="viewport" …> (ou avant <title>), colle exactement : <script src="https://cdn.tailwindcss.com"></script>.
  3. Supprime ou commente la ligne <link rel="stylesheet" href="estilos.css"> si tu ne veux plus du vieux fichier CSS.
  4. Enregistre, ouvre la page dans le navigateur, appuie sur F5 : Tailwind est actif ; tu peux maintenant écrire des classes dans class="…".

Étape 2 — index.html (accueil)

Cette page, c’est quoi ? La page d’accueil : le menu pour aller partout, ta photo, ton nom, le texte du profil, et en bas ton mail et tes liens.

Ce qu’on a utilisé pour coder

  • HTML : header, nav, a, main, img, footer.
  • Tailwind uniquement via l’attribut class="…" (pas de règles maison dans un .css pour cette page).
  • Des utilitaires typiques : flex, justify-center, bg-[#58008b], text-white, mx-auto, hover:underline, petites icônes svg avec h-4 w-4.

  3. Va sur tailwindcss.com → Flexbox et Backgrounds si tu veux changer une couleur : cherche le mot-clé, copie l’exemple dans ton class.

Mini examen — 30 preguntas con respuestas (pegadas)

  1. ¿Qué línea copias en el head para activar Tailwind por CDN?

    Respuesta: <script src="https://cdn.tailwindcss.com"></script>

  2. En la página de experiencia, ¿qué son details y summary?

    Respuesta: details es el bloque que se abre/cierra; summary es la línea visible donde haces clic.

  3. En un tabla, ¿qué hace rowspan="3"?

    Respuesta: hace que esa celda ocupe 3 filas (se “fusiona” verticalmente).

  4. Di dos clases Tailwind del menú que ayudan a poner los links en línea y centrarlos.

    Respuesta: por ejemplo flex + justify-center (también sirven items-center, flex-wrap).

  5. ¿En qué atributo HTML se escriben las clases Tailwind?

    Respuesta: en class="…".

  6. ¿Qué significa el prefijo bg- en Tailwind (ej. bg-white)?

    Respuesta: color (o estilo) de fondo del elemento.

  7. Según la tabla de capturas, ¿qué nombre tiene la imagen de certificados?

    Respuesta: images-04.png (en img/repaso/).

  8. ¿Cuál es la URL de la documentación oficial de Tailwind?

    Respuesta: https://tailwindcss.com/docs

  9. ¿Por qué hay que abrir repaso-examen-fr.html en la carpeta correcta para ver las imágenes?

    Respuesta: porque las rutas son relativas; si el HTML no está al lado de img/, el navegador no encuentra los archivos.

  10. ¿Para qué sirve hover: (ej. hover:underline)?

    Respuesta: aplica un estilo solo al pasar el ratón por encima.

  11. ¿Cuáles son 4 bloques típicos de una página (estructura HTML) que usaste?

    Respuesta: header, nav, main, footer.

  12. ¿Por qué en links externos se usa target="_blank" con rel="noopener noreferrer"?

    Respuesta: por seguridad (evitar window.opener) y comportamiento más seguro al abrir pestañas nuevas.

  13. ¿Qué hace mx-auto?

    Respuesta: centra un bloque horizontalmente (márgenes izquierda/derecha automáticos) si tiene ancho.

  14. Diferencia simple entre p- y m-.

    Respuesta: p- = padding (dentro); m- = margin (fuera).

  15. ¿Para qué sirve md:?

    Respuesta: para aplicar una clase solo a partir de un tamaño de pantalla “medio” (responsive).

  16. ¿Qué significa usar corchetes como bg-[#58008b] o w-[80%]?

    Respuesta: valor “a medida” (arbitrary value): color/ancho exacto no estándar.

  17. ¿Para qué sirve flex-wrap en el menú?

    Respuesta: permite que los enlaces bajen a otra línea si no cabe todo en una sola.

  18. Diferencia entre justify-center y items-center en flex.

    Respuesta: justify-… alinea en el eje principal; items-… en el eje secundario.

  19. ¿Para qué sirven gap-x-2 y gap-y-1?

    Respuesta: espacio entre elementos: horizontal (x) y vertical (y).

  20. ¿Por qué tu imagen tiene alt?

    Respuesta: accesibilidad (lectores de pantalla) y texto alternativo si la imagen no carga.

  21. ¿Qué hace text-center?

    Respuesta: centra el texto.

  22. ¿Qué hace font-bold?

    Respuesta: pone el texto en negrita.

  23. ¿Qué indican text-xs / text-xl?

    Respuesta: tamaño de letra (pequeño / grande).

  24. ¿Qué controlan leading-normal / leading-relaxed?

    Respuesta: el interlineado (espacio entre líneas).

  25. ¿Por qué usar overflow-x-auto en tablas o código?

    Respuesta: para poder hacer scroll horizontal sin romper el diseño en pantallas pequeñas.

  26. ¿Qué hacen rounded / rounded-lg?

    Respuesta: redondean las esquinas.

  27. ¿Para qué sirve border / border-2?

    Respuesta: poner un borde (más fino o más grueso).

  28. ¿Qué elemento suele contener el menú de navegación?

    Respuesta: <nav> (a menudo dentro de <header>).

  29. ¿Qué significa w-full?

    Respuesta: el elemento ocupa todo el ancho disponible.

  30. ¿Por qué usar max-w-3xl en vez de un ancho fijo?

    Respuesta: limita el ancho para mejor lectura, pero sigue siendo flexible en pantallas pequeñas.

  31. En una tabla, ¿por qué usar border-collapse?

    Respuesta: evita doble borde entre celdas (se “fusionan” las líneas).

  32. ¿Para qué sirve <strong>?

    Respuesta: marcar un texto como importante/enfatizado.

  33. ¿Para qué sirve un <svg> dentro de un link del menú?

    Respuesta: mostrar un icono vectorial (nítido) sin cargar una imagen externa.

  34. Diferencia entre inline-flex y flex.

    Respuesta: inline-flex se comporta como elemento “en línea”; flex como bloque.

  35. ¿Por qué usar no-underline y luego hover:underline?

    Respuesta: estilo limpio normal y feedback visual al pasar el ratón (mejor UX).

  36. Si el archivo se llama Idiomas.html (con I mayúscula), ¿por qué hay que copiar el nombre exacto en el href?

    Respuesta: porque si el nombre no coincide exactamente, el enlace puede fallar (según el sistema/servidor).

  37. En una frase: diferencia entre HTML y Tailwind en tu proyecto.

    Respuesta: HTML es la estructura y contenido; Tailwind es el estilo/maquetación con clases utilitarias.

    Comparación de CSS en HTML (lo más usado hoy + particularidad de cada uno)

    Hay muchas formas de “poner estilo” a una página. En 2026, lo más común es: CSS externo (archivos .css) y, en proyectos modernos, utility-first (Tailwind) o CSS-in-JS / módulos (sobre todo en React). En tu CV estás usando Tailwind por CDN, que es ideal para aprender y avanzar rápido.

    1) CSS inline (style="…")

    Particularidad: va pegado al elemento.

    Hoy se usa: poco (solo pruebas rápidas o casos muy puntuales).

    2) CSS interno (<style>)

    Particularidad: todo el CSS está dentro del mismo HTML.

    Hoy se usa: para páginas sueltas, demos o documentos.

    3) CSS externo (archivo .css + <link>)

    Particularidad: separación clara: HTML contenido, CSS estilo.

    Hoy se usa: muchísimo (lo más “estándar” para webs).

    4) Framework de componentes (Bootstrap)

    Particularidad: trae componentes ya diseñados + clases.

    Hoy se usa: aún bastante, sobre todo en proyectos rápidos o internos.

    5) Tailwind (utility-first)

    Particularidad: clases pequeñas en class="…" (como tu CV).

    Hoy se usa: muchísimo en proyectos modernos (especialmente con React/Next/Vite).

    6) CSS Modules (común en React)

    Particularidad: clases “locales” por componente (evita choques de nombres).

    Hoy se usa: bastante en apps React.

    ¿Cuál te conviene a ti ahora mismo?

    • Para aprender rápido y hacer páginas bonitas: Tailwind (CDN) como estás haciendo.
    • Para “lo más clásico” en cualquier trabajo: entiende bien CSS externo y selectores básicos.
    • Para empezar JavaScript: lo más importante es que tu HTML tenga id o clases para seleccionar, y luego JS cambia clases con classList.

    ¿Y dónde entran Flex y Grid?

    Flex y Grid NO son “tipos de CSS”. Son formas de maquetar (layout) dentro del CSS. Da igual si usas CSS inline/interno/externo o Tailwind: Flex/Grid siguen siendo CSS.

  <a href="#">Dos</a>
  <a href="#">Tres</a>
</nav>
      CSS (flex) 
      .menu {
  display: flex;
  justify-content: center;
  gap: 12px;
}
    • CSS inline: el CSS va dentro del HTML en style="…".
      inline 
      <button style="background:#58008b; color:white; padding:8px 12px;">
  Botón
</button>
    • CSS interno: el CSS va dentro del mismo HTML en un bloque <style>.
      HTML 
      <head>
  <style>
    .card { border:1px solid #ddd; padding:12px; }
  </style>
</head>

      .card {
  border: 1px solid #ddd;

    Resumen de CSS (tipos) + puente a JavaScript (nociones básicas)

    Esta parte es para repaso: entender cómo se aplica el CSS y cómo eso te ayuda a empezar con JavaScript (que “mueve” el HTML y cambia clases).

    A) Diferentes tipos / formas de usar CSS

    • CSS inline (en el HTML, atributo style).
      inline 
      <p style="color: #58008b; font-weight: bold;">Hola</p>

      Útil para una prueba rápida, pero se vuelve difícil de mantener.

    • CSS interno (en el mismo HTML con <style>).
      interno (style) 
      <head>
  <style>
    .titulo { color: #58008b; font-weight: 700; }
  </style>
</head>

      Bien para una sola página; para un sitio grande, conviene separar en archivo.

    • CSS externo (archivo .css + <link>).
      externo (link + css) 
      <link rel="stylesheet" href="estilos.css">

      Es lo más clásico: separa “contenido” (HTML) y “decoración” (CSS).

    • Framework CSS (ej. Bootstrap): te da componentes y clases ya hechas.

      Te ayuda a construir rápido, pero sigues “aprendiendo su forma” de hacer las cosas.

    • Utility-first (Tailwind): usas muchas clases pequeñas directamente en class="…".

      Eso es lo que hiciste en tu CV: cada palabra en class describe un detalle de estilo.

    B) ¿Cómo te ayuda CSS para empezar JavaScript?

    En proyectos reales, JavaScript casi siempre hace estas 3 cosas:

    • Leer el HTML (encontrar elementos).
    • Escuchar eventos (clic, escribir, enviar formulario).
    • Cambiar clases (y Tailwind hace el “estilo” automáticamente).

    C) Nociones básicas de JavaScript que deberías saber

    basicos.js 
    // 1) Variables
const titulo = "Hola";
let contador = 0;

// 2) Funciones
function saludar(nombre) {
  return "Hola " + nombre;
}

// 3) DOM: seleccionar elementos
const boton = document.querySelector("#btn");
const caja = document.querySelector(".caja");

// 4) Eventos
boton.addEventListener("click", () => {
  contador++;

  // 5) Cambiar clases (muy útil con Tailwind)
  caja.classList.toggle("hidden");
  caja.classList.add("bg-[#58008b]", "text-white");
});

    • Selectores: querySelector usa el mismo tipo de selectores que CSS (#id, .clase).
    • Eventos: click, input, submit…
    • classList: es la forma “normal” de poner/quitar clases (perfecto con Tailwind).
    Mini checklist para empezar JS en tu proyecto
    1. Dale id a lo que quieras controlar (ej. un botón y un div).
    2. Selecciona con document.querySelector.
    3. Escucha con addEventListener.
    4. Cambia clases con classList.add/remove/toggle.
