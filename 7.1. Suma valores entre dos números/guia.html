Guía JavaScript

Explicación línea por línea del archivo js/app.js del ejercicio «Calculadora de suma en un rango» (4 + 5 + 6 = 15).

Volver a la calculadora

¿Qué hace este programa?

El usuario escribe dos números enteros. Al pulsar Calcular, JavaScript suma todos los números desde el primero hasta el segundo, incluidos. Ese lenguaje es JavaScript (en el navegador), no Java.

1. Enlazar HTML → 2. Leer números → 3. Validar → 4. Bucle for → 5. Mostrar resultado

Código completo (js/app.js)

js/app.js
const inputIni = document.getElementById("ini");
const inputFin = document.getElementById("fin");
const btnCalcular = document.getElementById("btnCalcular");
const salida = document.getElementById("resultado");

function calcular() {
  let inicio = parseInt(inputIni.value, 10);
  let fin = parseInt(inputFin.value, 10);

  if (Number.isNaN(inicio) || Number.isNaN(fin)) {
    salida.textContent = "Escribe dos números enteros válidos.";
    return;
  }

  if (inicio > fin) {
    const temporal = inicio;
    inicio = fin;
    fin = temporal;
  }

  let suma = 0;

  for (let i = inicio; i <= fin; i++) {
    suma += i;
  }

  salida.textContent = suma;
}

btnCalcular.addEventListener("click", calcular);

Explicación línea por línea

Parte 1 — Enlazar la página con el HTML

1
const inputIni = document.getElementById("ini");

const crea una variable que no cambiará de elemento. document es toda la página. getElementById("ini") busca el primer input (id del HTML). Así puedes leer después lo que escribe el usuario con inputIni.value.

2
const inputFin = document.getElementById("fin");

Igual que la línea 1, pero para el segundo número. Hacen falta dos variables porque son dos cajas distintas.

3
const btnCalcular = document.getElementById("btnCalcular");

Guarda el botón «Calcular». Lo usarás para ejecutar la suma cuando el usuario haga clic.

4
const salida = document.getElementById("resultado");

Guarda el párrafo donde se muestra el resultado (15) o el mensaje de error. El usuario lo ve en la página, no en la consola.

Parte 2 — La función calcular

6
function calcular() {

function define un bloque de código con nombre. No se ejecuta al abrir la página; se ejecuta cuando el usuario pulsa el botón. Agrupa toda la lógica del ejercicio.

7-8
let inicio = parseInt(inputIni.value, 10);
let fin = parseInt(inputFin.value, 10);

.value es el texto del input. parseInt(..., 10) lo convierte en entero decimal. let porque inicio y fin cambian en cada clic. Sin esto solo tendrías texto ("4"), no números para sumar.

11-14
if (Number.isNaN(inicio) || Number.isNaN(fin)) { ... return; }

Si el campo está vacío o no es número, parseInt da NaN. Se muestra un mensaje de error y return detiene la función para no entrar al bucle.

16-20
if (inicio > fin) { intercambio con temporal }

Si escriben 6 y 4, el bucle necesita inicio ≤ fin. Se intercambian los valores para que 6 y 4 den el mismo resultado que 4 y 6 (15).

23
let suma = 0;

Acumulador: empieza en cero. En cada vuelta del for irás sumando (4, luego 9, luego 15).

25-27
for (let i = inicio; i <= fin; i++) { suma += i; }

Bucle for: i va de inicio a fin. i <= fin controla cuándo parar. i++ sube 1 cada vuelta (incremento). suma += i añade cada número al total.

29
salida.textContent = suma;

Escribe el número final en el <p id="resultado"> de la página.

35
btnCalcular.addEventListener("click", calcular);

Fuera de la función: al cargar la página se registra «cuando hagan clic, ejecuta calcular». Debe estar fuera de calcular, si no el botón no funciona al inicio.

Dibujo del bucle for (ejemplo 4 y 6)

  inicio = 4          fin = 6
     ↓                   ↓
     4  ──→  5  ──→  6  ──→  7 (aquí para: 7 <= 6 es falso)

  suma empieza en 0

  Vuelta 1:  i = 4   →  suma = 0 + 4 = 4      →  i++  →  i = 5
  Vuelta 2:  i = 5   →  suma = 4 + 5 = 9      →  i++  →  i = 6
  Vuelta 3:  i = 6   →  suma = 9 + 6 = 15     →  i++  →  i = 7
  Comprueba: i = 7, ¿7 <= 6? NO → sale del for

  Resultado en pantalla: 15

10 preguntas y respuestas

1 ¿Por qué usamos getElementById al inicio del archivo?

Para guardar una referencia a cada parte del HTML (inputs, botón, párrafo de resultado). Si no lo hicieras, tendrías que buscar en el documento cada vez. Así el código es más claro y coincide con los id de tu index.html (ini, fin, btnCalcular, resultado).

2 ¿Qué diferencia hay entre const y let?

const se usa cuando la variable sigue apuntando al mismo elemento (inputIni siempre es el mismo input). let se usa cuando el valor cambia (inicio, fin, suma, i en el bucle). No puedes reasignar const a otro elemento, pero sí leer inputIni.value que cambia cada vez.

3 ¿Por qué parseInt lleva un 10 al final?

El 10 indica base decimal (números enteros normales: 4, 6, 15). El input devuelve texto; parseInt convierte "4" en el número 4 para poder sumar en el for.

4 ¿Qué es NaN y por qué lo comprobamos?

NaN significa «Not a Number» (no es un número). Aparece si el campo está vacío o tiene algo que no se puede convertir. Number.isNaN evita que el programa falle y muestra un mensaje claro al usuario.

5 ¿Para qué sirve return dentro del if?

return termina la función calcular en ese momento. Si no hay números válidos, no debe ejecutarse el for ni mostrarse un resultado incorrecto. Es como decir «ya terminé, no sigas».

6 ¿Por qué intercambiamos inicio y fin si inicio > fin?

El bucle for está pensado para ir de un número pequeño a uno grande (i <= fin). Si el usuario escribe 6 y 4, sin intercambiar el bucle no sumaría bien. Tras el intercambio, 6 y 4 se tratan como 4 y 6 y el resultado es 15.

7 ¿Qué hace i++ en el for?

i++ es el operador de incremento: suma 1 a i después de cada vuelta. Pasa de 4 a 5, de 5 a 6, etc. La profe lo pide como parte del trabajo con operadores de incremento/decremento.

8 ¿Qué significa suma += i?

Es lo mismo que suma = suma + i. En cada vuelta añades el valor actual de i al acumulador. Así construyes 4 + 5 + 6 paso a paso hasta llegar a 15.

9 ¿Por qué addEventListener debe estar fuera de calcular?

addEventListener se ejecuta una vez al cargar la página y registra el clic. Si estuviera dentro de calcular, nadie llamaría a calcular al inicio y el botón no haría nada. Además, si estuviera dentro, cada clic añadiría otro listener duplicado.

10 ¿Para qué sirve la consola del navegador (F12)?

Para ti como programadora: ver errores en rojo, probar document.getElementById("ini").value o console.log mientras aprendes. El usuario final ve el resultado en salida.textContent en la página; la consola no la necesita para usar la calculadora.
