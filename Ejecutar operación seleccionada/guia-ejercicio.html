Guía de estudio

Ejercicio: ejecutar una operación seleccionada

Esta guía resume el objetivo del ejercicio y explica, línea por línea, tu index.html y tu js/app.js. Después tienes 15 FAQs y el mismo contenido como quiz tipo ABC; las respuestas están ocultas en elementos <details>.

¿En qué consistía el ejercicio?

Consistía en crear una página web donde el usuario escribe dos números, elige una operación en un desplegable (<select>) y pulsa Calcular. El programa debe aplicar solo la operación seleccionada y mostrar el resultado.

En tu solución actual, la validación básica se hace con mensajes (alert) cuando falta un número o la división no es válida (dividir entre cero). El resultado visible queda escrito en un párrafo con id="resultado".

Qué tecnologías usa tu proyecto

  • HTML: estructura del formulario (inputs, select, botón y lugar donde se muestra el resultado).
  • Tailwind (CDN): clases para layout y estilo.
  • JavaScript (DOM): escuchar el click del botón, leer valores y escribir el resultado.

Pasos que ejecuta tu JavaScript (mentalmente)

  1. Obtiene referencias a los elementos del DOM por id.
  2. Comprueba que todos existan; si falta algo, muestra una alerta y evita ejecutar lo demás.
  3. Cuando haces click en Calcular: lee y convierte los valores, valida entradas básicas y calcula con switch.
  4. Escribe el resultado en pantalla usando textContent.

15 preguntas y respuestas

1) ¿Qué problema resuelve este ejercicio?

Conectar una interfaz HTML simple con JavaScript para que el usuario ejecute una operación aritmética elegida y vea el resultado.

2) ¿Por qué los inputs tienen id?

Para que JavaScript pueda localizarlos con document.getElementById de forma única y estable.

3) ¿Qué es type="number"?

Indica al navegador que el campo es numérico y suele mostrar controles extra; aun así convierte la lectura a número en JS.

4) ¿Para qué sirve el placeholder?

Es una pista visual dentro del campo; no sustituye un valor real y no es el texto “oficial” del campo.

5) ¿Qué representa <select> y <option value>?

El select es el desplegable; cada option tiene texto visible y un value que es lo que lee tu JS ("+", etc.).

6) ¿Por qué el botón es type="button"?

Evita comportamientos de envío típicos si el botón viviera dentro de un formulario; aquí ejecuta solo JS con el click.

7) ¿Qué significa grid-cols-4 en tu HTML?

Organiza horizontalmente cuatro controles del mismo tamaño dentro de una cuadrícula de 4 columnas.

8) ¿Qué diferencia hay entre cargar Tailwind CDN y tener CSS manual?

Tailwind proporciona utilidades rápidas con clases (class="...") en lugar de escribir muchas reglas CSS a mano.

9) ¿Por qué se usa Number(...)?

Porque input.value es texto; necesitas número para sumar/multiplicar con operadores matemáticos.

10) ¿Para qué sirve Number.isFinite?

Comprueba que el resultado de la conversión es un número válido finito antes de calcular.

11) ¿Por qué se validan strings vacíos con trim()?

Para rechazar inputs “vacíos” aunque tengan espacios accidentalmente escritos por el usuario.

12) ¿Qué papel juega switch?

Selecciona qué operación aplicar comparando contra los valores esperados ("+", "-", etc.).

13) ¿Por qué dividir entre cero es un caso especial?

En JavaScript puedes obtener Infinity u otros resultados no deseados; lo normal es avisar y parar.

14) ¿Por qué se usa textContent y no innerHTML?

textContent escribe texto plano sin interpretar HTML, lo que reduce riesgos si el contenido proviene del usuario.

15) ¿Por qué el script va al final del body?

Así el HTML ya existe cuando el JS busca elementos por id (evita null por orden de carga).
