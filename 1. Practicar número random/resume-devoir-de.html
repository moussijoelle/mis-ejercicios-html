1. Fichiers du projet et rôle de chacun

Ton mini-projet repose sur trois éléments principaux :

  • index.html — C'est le document que le navigateur charge en premier. Il décrit la structure de la page (titres, texte, images, bouton) et indique où trouver les styles (Tailwind via CDN) et le script (app.js). Le HTML ne « calcule » pas le hasard : il se contente d'afficher des éléments et de donner des identifiants (id) pour que le JavaScript puisse les retrouver.
  • app.js — C'est le fichier de comportement. Il contient la fonction tirarDado qui tire un nombre entre 1 et 6, construit le chemin du fichier image, et met à jour l'attribut src de l'image du résultat. Il contient aussi le code qui enregistre l'écoute du clic sur le bouton une fois la page prête.
  • Dossier img/ — Il contient les six images dado1.png à dado6.png. Le navigateur les charge lorsque l'URL du src pointe vers ces fichiers (par exemple img/dado4.png). Si un chemin est faux ou si un fichier manque, l'image ne s'affiche pas correctement.

2. Vue d'ensemble du fonctionnement

Quand tu ouvres index.html, le navigateur construit une représentation interne de la page appelée le DOM (Document Object Model). Chaque balise devient un nœud : par exemple ton bouton et ton image résultat sont des nœuds auxquels le JavaScript peut accéder par leur id.

Ensuite le navigateur charge et exécute app.js. Le bloc DOMContentLoaded attend que le HTML soit prêt, puis attache un écouteur d'événement sur le bouton : « à chaque clic, appelle tirarDado ». Quand tu cliques, tirarDado calcule un entier aléatoire entre 1 et 6, fabrique une chaîne de caractères qui correspond au chemin du fichier PNG (concaténation), et assigne cette chaîne à document.getElementById("imagenDado").src. Le navigateur voit que le src a changé et charge l'image correspondante : tu vois une nouvelle face du dé.

3. HTML (index.html) : explications détaillées

3.1 En-tête du document (<head>)

<!DOCTYPE html> indique au navigateur qu'il s'agit d'un document HTML5 moderne. <html lang="es"> enveloppe toute la page ; lang="es" correspond au texte espagnol visible.

<meta charset="UTF-8"> garantit que les accents s'affichent correctement. viewport aide au rendu sur mobile.

Le script Tailwind et le bloc <style type="text/tailwindcss"> stylisent la page (bouton, image du dé avec #imagenDado, fond centré). <title>Dado al azar</title> = titre de l'onglet.

3.2 Corps (<body>) — contenu visible

class="min-h-dvh flex …" centre tout verticalement et horizontalement sur un fond gris clair.

<main> regroupe le contenu principal :

  • <h1>BUENA SUERTE</h1> — titre affiché en vert.
  • <img id="imagenDado" src="img/dado1.png"> — l'image du dé. C'est elle dont le JavaScript change le src pour montrer les faces 1 à 6.
  • <button id="btnLanzarDado"> — le bouton « Lanzar dado ». Pas de onclick dans le HTML : le clic est branché dans app.js.

3.3 Chargement de app.js

<script src="app.js"></script> en fin de body : le bouton et l'image existent déjà quand le script s'exécute.

5. JavaScript (app.js) : explications détaillées

5.1 La fonction tirarDado

Une fonction regroupe des instructions réutilisables. Ici, elle encapsule tout ce qui doit se passer quand l'utilisateur « lance le dé ».

const numeroRandom = Math.ceil(Math.random() * 6); — Math.random() renvoie un nombre décimal pseudo-aléatoire dans l'intervalle [0, 1[ (zéro inclus, un exclu). Le multiplier par 6 donne un nombre dans [0, 6[ : on a ainsi « la bonne taille » d'intervalle pour six faces. Math.ceil arrondit vers le haut à l'entier le plus proche : presque tout ce qui tombe entre 0 (exclu) et 6 devient un entier de 1 à 6, comme les faces d'un dé. C'est la variante souvent demandée en cours à la place de Math.floor(Math.random() * 6) + 1 : avec floor, on obtient d'abord 0…5 puis le + 1 décale vers 1…6 ; avec ceil, on n'a pas besoin de ce + 1 pour la même idée (mapper l'intervalle continu sur six entiers). Remarque théorique : si Math.random() renvoyait exactement 0, Math.ceil(0) vaudrait 0 (pas une face valide) ; en pratique c'est extrêmement rare ; pour un exercice scolaire l'important est de comprendre ceil vs floor.

const rutaImagen = "img/dado" + numeroRandom + ".png"; — En JavaScript, l'opérateur + entre une chaîne et un nombre convertit le nombre en texte et concatène. Ainsi, si numeroRandom vaut 4, la variable contient la chaîne img/dado4.png, qui est l'URL relative correcte vers ton fichier dans le dossier img.

document.getElementById("imagenDado").src = rutaImagen; — document représente la page chargée. getElementById renvoie l'élément DOM correspondant. La propriété src d'une image est l'adresse de la ressource affichée ; lui assigner une nouvelle chaîne demande au navigateur de charger la nouvelle image (souvent avec mise en cache si le fichier a déjà été vu).

5.2 L'écouteur DOMContentLoaded

document.addEventListener("DOMContentLoaded", () => { … }); enregistre une fonction (ici une fonction fléchée) qui s'exécute une fois le document HTML analysé et le DOM construit. Cela évite d'appeler getElementById("btnLanzarDado") trop tôt, avant que le bouton n'existe.

5.3 L'écouteur de clic sur le bouton

.addEventListener("click", tirarDado) attache la fonction tirarDado comme gestionnaire du clic. On écrit tirarDado sans parenthèses pour passer la référence de la fonction (le navigateur l'invoquera au moment du clic). Écrire tirarDado() ici appellerait la fonction immédiatement au chargement, ce qui n'est pas souhaité.

7. Voir les autres faces du dé — et le « d » (<details>)

Les faces du dé (exercice) : une seule balise <img id="imagenDado">. Au clic, JavaScript calcule 1–6 et remplace src par img/dado + nombre + .png. Le navigateur charge alors dado2.png, dado5.png, etc. — c'est ainsi que tu « vois les autres faces » sans six images en même temps sur la page du jeu.

Le « d » pour cacher les réponses du test : la balise HTML <details> (détails) avec <summary> masque le texte jusqu'au clic. Ce fichier récap et guia.html utilisent ce mécanisme pour les questions : tu ne vois la réponse détaillée que si tu ouvres le bloc. Ce n'est pas la touche D du clavier.

8. Quinze questions — réponses cachées (clique pour ouvrir)

Réponds d'abord sans ouvrir. Puis clique sur chaque question pour la correction détaillée.

1. En quoi consiste l'exercice ?

Créer une page avec un dé et un bouton. Au clic, un nombre de 1 à 6 est tiré au hasard et l'image affichée change pour montrer la bonne face (img/dadoX.png). On apprend le hasard en JavaScript et la modification du src d'une image.

2. HTML vs JavaScript ?

HTML = structure (titre, image, bouton). JavaScript = action au clic (nombre + nouveau src). Sans JS, l'image reste sur dado1.png.

3. Comment voir les autres faces du dé ?

On ne cache pas six images : on change l'adresse src de imagenDado. Exemple : tirage 4 → img/dado4.png. Le navigateur charge ce fichier.

4. Pourquoi Math.ceil(Math.random() * 6) ?

random() donne un décimal entre 0 et 1. × 6 → intervalle adapté. ceil arrondit vers le haut → entiers 1 à 6. Variante équivalente : Math.floor(Math.random() * 6) + 1.

5. Qu'est-ce que la concaténation ?

"img/dado" + numeroRandom + ".png" assemble des morceaux de texte. Si le nombre est 3, on obtient la chaîne img/dado3.png.

6. Rôle des id ?

Identifiants uniques pour que getElementById retrouve l'image et le bouton sans ambiguïté.

7. Pourquoi tirarDado sans () dans addEventListener ?

Sans parenthèses, on passe la fonction au navigateur pour qu'il l'appelle au clic. Avec tirarDado(), elle s'exécuterait tout de suite au chargement.

8. À quoi sert DOMContentLoaded ?

Attendre que le HTML soit prêt avant d'attacher l'écouteur au bouton, pour éviter « élément introuvable ».

9. Pourquoi img/ dans le chemin ?

Les PNG sont dans le sous-dossier img. Le chemin relatif doit correspondre à l'emplacement réel des fichiers.

10. Java vs JavaScript ?

Deux langages différents. Ici c'est JavaScript dans le navigateur (app.js).

11. Comment déboguer si rien ne marche ?

F12 → Console (erreurs JS) et Réseau (images 404 si fichier manquant ou mauvais nom).

12. Le hasard est-il « vrai » ?

Math.random() est pseudo-aléatoire : suffisant pour un exercice, pas pour la cryptographie.

13. Qu'est-ce que le DOM ?

Arborescence d'objets créée par le navigateur à partir du HTML. Modifier .src sur le nœud image change ce qu'on voit.

14. À quoi sert <details> (le « d ») ici ?

Balise HTML qui masque le contenu jusqu'au clic sur <summary>. Utilisée pour cacher les réponses du test dans ce récap et dans guia.html. Ce n'est pas la touche D du clavier.

15. Que livrer pour le devoir ?

index.html, app.js, dossier img/ avec les six PNG. guia.html et resume-devoir-de.html sont des aides personnelles.
