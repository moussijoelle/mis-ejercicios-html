Ir al programa

Encontrar el primer número primo

Explicaciones, código línea por línea y test de 15 preguntas. Haz clic en cada pregunta para ver la respuesta cuando quieras corregir.

1. En qué consiste el ejercicio

Crea un programa que revise los números en un array hasta que encuentre el primer número primo del mismo. Aprovecha la función número primo del ejercicio anterior. Pista: do-while.
  1. Tienes una lista ordenada (array), por ejemplo: 4, 9, 7, 10, 2.
  2. Empiezas por el primero y preguntas: ¿es primo?
  3. Si no → siguiente. Si sí → paras y ese es el resultado.
  4. No es el primo más pequeño de la lista, sino el primero en orden.

Ejemplo: 4, 9, 7, 10, 2 → respuesta 7 (no 2).

6. Test — 15 preguntas

Responde primero. Luego haz clic en la pregunta para ver la respuesta.

1. ¿Qué es un array en este ejercicio?

Un array guarda varios valores en orden, con posiciones desde 0. El usuario escribe 4, 9, 7, 10, 2 y JavaScript lo convierte en [4, 9, 7, 10, 2] para revisar cada número. Sin array solo tendrías un número; aquí necesitas una lista.

2. ¿Qué significa «el primer número primo»?

No es el primo más pequeño, sino el primero en orden del array. En 4, 9, 7, 10, 2 la respuesta es 7, no 2. El programa se detiene al encontrar el primero válido.

3. ¿Por qué el 1 no es primo?

if (num <= 1) return false. En el curso el primo debe ser mayor que 1. Coincide con el ejercicio 7.4.

4. ¿Qué hace % en num % divisor === 0?

Devuelve el resto. Si es 0, hay división exacta → no es primo. Ejemplo: 10 % 2 === 0 → 10 no es primo.

5. ¿Para qué sirve esPrimo(num)?

Devuelve true o false. Comprueba ≤ 1, luego divisores con for. Reutilizada del ejercicio anterior (DRY).

6. Diferencia entre while y do-while

while puede no ejecutarse nunca. do-while ejecuta al menos una vez. La condición primerPrimo === null && indice < numeros.length controla cuándo seguir.

7. ¿Qué es primerPrimo = null?

null = todavía no hay primo encontrado. Distinto de 0. Si termina el bucle sin encontrar, sigue siendo null.

8. ¿Qué hacen split y map?

split(',') parte el texto. trim() quita espacios. Number() convierte a número. Resultado: array numérico.

9. ¿Por qué preventDefault()?

Evita que el formulario recargue la página. Solo corre tu JavaScript.

10. Lista 2, 3, 5 — ¿resultado?

2. Es el primer primo en orden; no llega a 3 ni 5.

11. Lista 4, 6, 8 — ¿qué pasa?

Ninguno es primo. Devuelve null y muestra: no hay ningún número primo en la lista.

12. ¿Qué es DRY?

Don't Repeat Yourself: la lógica de primo está solo en esPrimo; encontrarPrimerPrimo solo la llama.

13. ¿Para qué sirven los id?

Identifican elementos para getElementById. Si HTML y JS no coinciden, el programa falla.

14. ¿Qué hace resultado.innerHTML?

Escribe HTML dentro del div resultado. Usa plantillas `...${primerPrimo}...` para insertar el número.

15. Si escribe hola, 7 — ¿qué pasa?

hola → NaN. La validación lo detecta, muestra error y no llama a encontrarPrimerPrimo.

7. Cómo estudiar

  1. Lee las secciones 1 y 2.
  2. Abre index.html y js/app.js en paralelo.
  3. Haz el test sin abrir las respuestas.
  4. Abre solo las que quieras corregir.
