HTML — index.html línea a línea

Línea 1

<!DOCTYPE html>

Declara que el documento es HTML5. Obligatorio para que el navegador interprete bien la página.

Línea 2

<html lang="es" data-theme="light">

Raíz del documento. lang="es" = español. data-theme="light" = tema claro DaisyUI (el JS puede cambiarlo a night).

Línea 3

<head>

Abre la cabecera: meta, título, CSS y scripts. No es lo visible del carrusel.

Línea 4

<meta charset="UTF-8">

Codificación UTF-8 para tildes, ñ y símbolos (›, «).

Línea 5

<meta name="viewport" ...>

Adapta la página a móviles: ancho = pantalla, zoom inicial 1.

Línea 6

<title>Mapa del Top 20...</title>

Título de la pestaña del navegador.

Línea 7

(vacía)

Solo separa bloques en el editor.

Línea 8

<link rel="icon" ... favicon-32x32.png>

Favicon: icono pequeño de la pestaña.

Línea 9

(vacía)

Separador.

Línea 10

daisyui.css

Componentes DaisyUI (btn, card, badge, swap…). Requiere internet.

Línea 11

themes.css

Temas de color extra (light, night, etc.). Complementa la línea 10.

Línea 12

@tailwindcss/browser@4

Tailwind 4 en el navegador: clases como flex, max-w-6xl, rounded-2xl.

Línea 13

font-awesome ... all.min.css

Iconos Font Awesome (opcional; en tu página casi no se usan).

Líneas 14-15

preconnect fonts.googleapis.com / gstatic

Conexión anticipada a Google Fonts para cargar más rápido.

Línea 16

Urbanist wght 400;600;800

Descarga la fuente Urbanist que aplicas en el <style>.

Línea 17

(vacía)

Separador.

Línea 18

<style>

CSS propio de la página (colores del título y slide Info).

Línea 19

* { font-family: 'Urbanist', ... }

Toda la página usa Urbanist.

Líneas 21-27

#titulo-principal { color: #1f2937; } ... night ... #38bdf8

Título del header gris oscuro en día; azul claro en tema noche.

Líneas 29-35

#intro-subtitulo en slide-info

En la diapositiva final «Info», el subtítulo es verde (#00b300) de día y azul de noche.

Líneas 37-38

</style> </head>

Cierra estilos y cabecera.

Línea 40

<body class="bg-base-300 min-h-screen grid ... place-items-center ...">

Cuerpo: fondo gris, altura mínima pantalla, centra el contenido con CSS Grid.

Línea 42

<div class="relative w-full max-w-6xl">

Contenedor del carrusel + reloj. relative permite posicionar el reloj al lado sin achicar el carrusel.

Líneas 43-53

<aside id="panel-reloj"> ... <div id="reloj">

Panel del reloj. En móvil va arriba; en pantalla grande (lg:) se coloca a la izquierda del carrusel (absolute right-full). Solo la caja negra con la hora.

Línea 50

class="w-[7.5rem] ... bg-black ... font-mono text-lg"

Reloj pequeño: ancho fijo, fondo negro, números claros, fuente monoespaciada.

Línea 51

aria-live="polite"

Accesibilidad: lectores de pantalla anuncian cambios de hora sin interrumpir.

Línea 55

<div id="main-card" class="card ... max-w-6xl">

Tarjeta principal del carrusel: mismo ancho máximo que antes (6xl).

Línea 57

<header id="page-header">

Cabecera: logo, título, play/pausa y tema día/noche.

Líneas 59-66

<img id="logo-header" ...>

Logo del proyecto. alt describe la imagen. Clases Tailwind controlan tamaño responsive.

Líneas 67-69

<h1 id="titulo-principal">

Título visible del mapa. El color lo define tu CSS en <style>.

Líneas 70-76

<button id="btn-play-pause"> ▶

Botón autoplay del carrusel (temporizador de fotos). El JS cambia a ⏸ al activar.

Líneas 79-87

<label class="swap"> ... theme-toggle ... SVG sol/luna

Interruptor de tema: checkbox oculto de DaisyUI theme-controller + dos SVG que rotan (swap).

Línea 90

<section id="carousel-slide">

Zona del carrusel (figura + datos del anime).

Línea 91

<figure id="carousel-figure">

Marco de la imagen o textos de portada/Info. Altura mínima ~50–62vh.

Líneas 92-101

<div id="carousel-intro"> intro-titulo, subtitulo, extra, fuente, hint

Bloque de texto para Portada e Info. Los párrafos empiezan vacíos; app.js los rellena. intro-hint = «Pulsa ›…».

Líneas 103-108

<img id="carousel-img" class="... hidden" src="">

Imagen del anime. Oculta al inicio; el JS pone src y quita hidden en slides de anime.

Líneas 110-119

botones « ‹ › » con btn-first, prev, next, last

Navegación manual. pointer-events-none en el contenedor y auto en botones para que la imagen no bloquee clics.

Líneas 122-127

<article id="anime-info"> puesto, nombre, descripcion, lista

Datos bajo la imagen: badge de puesto, nombre, descripción y lista Estilo/Episodios (rellenados por JS).

Líneas 130-132

Cierres de main-card y contenedor relative

Cierra la tarjeta y el wrapper.

Línea 134

<script src="js/app.js"></script>

Carga todo el JavaScript: carrusel, tema, reloj y autoplay en un solo archivo.

Líneas 135-136

</body> </html>

Fin del documento.

JavaScript — js/app.js línea a línea

Las líneas 27–228 son datos de cada anime (misma estructura repetida). Abajo se explica el patrón una vez.

Líneas 1–25

const ... = document.getElementById("...");

Guardas referencias a elementos del HTML para no buscarlos en cada función. Incluye reloj, botones, textos del carrusel y tema.

Líneas 27–228

const misAnimes = [ { Portada: {...} }, { "Anime 1": {...} }, ... { Info: {...} } ];

Array con todas las diapositivas. Cada elemento es un objeto con una clave (Portada, Anime N, Info). Portada tiene esIntro; Info tiene esInfo, párrafo y fuente; los demás tienen nombre, estilo, episodios, descripción e imagen.

Línea 231

let indexActual = 0;

Índice de la diapositiva actual (0 = portada).

Línea 232

let intervalo = null;

ID del temporizador del autoplay del carrusel. null = parado.

Línea 233

let intervaloReloj = null;

ID del temporizador del reloj (separado del carrusel).

Línea 234

let sentidoAutoplay = 1;

Dirección del autoplay: 1 = adelante, -1 = atrás (rebota al llegar al final).

Línea 235

let carruselIniciado = false;

Evita registrar dos veces los eventos de clic.

Línea 237

const CLAVE_TEMA = "tema-anime-carrusel";

Nombre de la clave en localStorage para recordar light/night.

Líneas 239–249

CLASES_CARD_NORMAL, COMPACTA, HEADER, LOGO...

Listas de clases Tailwind para layout «grande» (portada/Info) o «compacto» (anime con foto).

Líneas 251–259

function alternarGrupo(elemento, activar, clasesOn, clasesOff)

Quita un grupo de clases y pone otro en un elemento (cambiar estilos sin escribir HTML de nuevo).

Líneas 261–268

aplicarEstiloIntro()

Aplica clases «normales» (más padding, logo grande) para portada e Info.

Líneas 270–276

aplicarEstiloAnime()

Aplica clases compactas para dejar más espacio a la imagen del anime.

Líneas 278–283

aplicarTema(tema)

Pone data-theme en <html>, marca el checkbox y guarda en localStorage.

Líneas 285–288

restaurarTema()

Lee el tema guardado al cargar (o usa light por defecto).

Líneas 290–294

function irA(indice)

Cambia de slide con módulo para que al pasar del último vuelva al primero (y al revés). Llama actualizarPantalla().

Líneas 296–298

ultimoIndice()

Devuelve el índice del último elemento del array.

Líneas 300–331

obtenerSlide(indice)

Lee misAnimes[indice], saca la clave y datos, y devuelve un objeto uniforme según sea intro, info o anime.

Líneas 333–347

mostrarEnlaceFuente(fuente)

Crea un <a> en intro-fuente o lo oculta si no hay enlace.

Líneas 349–356

lineasCaracteristicas(slide)

Devuelve array con textos «Estilo: …» y «Episodios: …».

Líneas 358–365

mostrarMarcoTexto()

Muestra el div de texto, oculta imagen y bloque anime-info (portada/Info).

Líneas 367–417

actualizarPantalla()

Función central: según el slide actual, rellena textos, imagen, clases CSS y visibilidad de bloques.

Líneas 419–425

siguiente() / anterior()

Avanzan o retroceden una diapositiva llamando a irA.

Líneas 427–437

tickAutoplay()

Se ejecuta cada 3 s con setInterval: invierte sentido en los extremos y llama irA.

Líneas 439–447

detenerAutoplay()

clearInterval(intervalo), pone intervalo a null y botón en ▶ verde.

Líneas 449–455

iniciarAutoplay()

setInterval(tickAutoplay, 3000) y botón en ⏸ rojo. Este es el temporizador del carrusel (fotos).

Líneas 457–470

actualizarReloj()

Lee new Date(), formatea HH:MM:SS con cero delante si hace falta y escribe en #reloj.

Líneas 472–477

iniciarReloj()

setInterval(actualizarReloj, 1000) — reloj automático cada segundo.

Líneas 479–502

iniciarCarrusel()

Engancha clics en navegación, play/pausa y tema; restaura tema y pinta la primera pantalla.

Líneas 504–507

DOMContentLoaded → iniciarReloj(); if (mainCard) iniciarCarrusel();

Cuando el HTML está listo, arranca reloj y carrusel (solo carrusel si existe #main-card).

15 preguntas para repasar

Pulsa cada pregunta para ver la respuesta.

1. ¿Qué diferencia hay entre el reloj y el temporizador del carrusel?

Ver respuesta

El reloj muestra la hora real con setInterval(..., 1000). El temporizador del carrusel cambia de foto cada 3 s con setInterval(tickAutoplay, 3000) al pulsar ▶. Misma función (setInterval), distinto uso.

2. ¿Para qué sirve clearInterval en este proyecto?

Ver respuesta

Para parar un intervalo activo. En el carrusel, detenerAutoplay() hace clearInterval(intervalo) al pulsar ⏸. Sin eso, el carrusel seguiría avanzando solo.

3. ¿Qué hace document.getElementById?

Ver respuesta

Busca en el HTML el elemento con ese id y devuelve una referencia para cambiarlo con JS (texto, clases, eventos).

4. ¿Dónde están los datos de los 20 animes?

Ver respuesta

En el array misAnimes dentro de app.js. El HTML solo tiene contenedores vacíos; el JS rellena nombre, imagen, etc.

5. ¿Qué es indexActual?

Ver respuesta

Variable con el número de la diapositiva actual (0 = portada, 1 = Anime 1, …, último = Info).

6. ¿Cómo funciona el botón ▶ / ⏸?

Ver respuesta

Si intervalo === null, llama iniciarAutoplay() (setInterval 3 s). Si no, llama detenerAutoplay() (clearInterval).

7. ¿Qué hace data-theme en la etiqueta <html>?

Ver respuesta

DaisyUI lee ese atributo para aplicar colores del tema light o night. Lo cambia aplicarTema() al marcar sol/luna.

8. ¿Para qué se usa localStorage aquí?

Ver respuesta

Para recordar si elegiste tema claro u oscuro. Al recargar, restaurarTema() lee esa preferencia.

9. ¿Por qué carousel-img tiene hidden al inicio?

Ver respuesta

En portada e Info no hay foto de anime; se muestra texto. El JS quita hidden solo cuando el slide es un anime con imagen.

10. ¿Qué hace irA(indice) con números negativos o muy grandes?

Ver respuesta

Usa módulo ((indice % total) + total) % total para dar la vuelta: antes del 0 va al último, después del último vuelve al 0.

11. ¿Qué es tickAutoplay?

Ver respuesta

Función que se ejecuta cada 3 segundos en autoplay: ajusta sentidoAutoplay en los extremos y llama irA para cambiar de slide.

12. ¿Por qué hay dos variables intervalo e intervaloReloj?

Ver respuesta

Para no mezclar dos tareas: una intervalo controla el carrusel (3 s), otra el reloj (1 s). Cada una se para con su propio clearInterval si hiciera falta.

13. ¿Qué hace la clase sr-only en el título de portada?

Ver respuesta

Oculta visualmente el texto pero lo deja disponible para lectores de pantalla (accesibilidad).

14. ¿Qué CDN cargas en el HTML y por qué necesitas internet?

Ver respuesta

DaisyUI, themes, Tailwind browser y Google Fonts vienen de internet. Sin conexión esos estilos no cargan (el JS local sí funciona).

15. ¿En qué slide aparece el enlace a Wikipedia?

Ver respuesta

En la diapositiva Info (última), mediante mostrarEnlaceFuente(slide.fuente) con la URL y texto definidos en misAnimes.
