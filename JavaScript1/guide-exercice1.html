Ce qu’on construit (et pourquoi)

Page 1 / 9

FR Avant de lire le guide : voici la consigne et le résultat attendu.

Consigne (très simple) : tu écris 2 nombres dans un formulaire, tu cliques sur le bouton Aritmética, et la page affiche 4 résultats : +, −, ×, ÷.

Idée : c’est comme une petite calculatrice qui parle avec des phrases (“la suma de … es …”).

Pourquoi “Viewport” et “Go Live” peuvent être différents : le mode viewport simule un téléphone (petite largeur), donc les règles CSS @media s’activent (layout mobile). Go Live ne change pas le style : il sert juste la page, mais tu peux voir des différences si la largeur/zoom n’est pas la même.

Pour l’audio : la voix dépend de ton navigateur et des voix installées sur Windows. Si une voix FR/ES n’est pas disponible, le navigateur peut choisir une autre voix.

Les étapes importantes (dans l’ordre) :

1) Je mets des id dans le HTML → 2) Je récupère avec getElementById → 3) Je lis .value → 4) Je convertis en nombre (parseInt) → 5) Je fais les 4 calculs → 6) Je montre les phrases dans les paragraphes.

Astuce : si tu suis ces 6 étapes, tu peux refaire l’exercice sans te perdre.

Les morceaux HTML importants

Page 2 / 9

En HTML, tu as donné des id à des éléments. Un id est comme le “nom unique” d’un objet.

Exemple de la vie : dans une classe, si 2 élèves s’appellent “Alex”, c’est confus. Mais si chaque élève a un numéro unique, tu peux trouver la bonne personne vite. L’id fait ça. 

<input type="number" id="numeroa">
<input type="number" id="numerob">
<button type="button" onclick="calcular()">Aritmética</button>

<p id="suma"></p>
<p id="resta"></p>
<p id="multiplicacion"></p>
<p id="division"></p>

NB : Pour tout ce qui est action (calculer, afficher un résultat, changer du texte, réagir au clic), c’est JavaScript.

Mais un bouton peut quand même avoir un effet “simple” sans JS :

  • Si c’est un lien (<a href="...">) → ça navigue
  • Si c’est un formulaire (<button type="submit">) → ça envoie le formulaire
  • Avec CSS → il peut changer de couleur au survol (mais ça ne “calcule” rien)
Mots clés pour l’examen (ES) : id, input, botón, onclick, párrafo.

Étape 1 — JS “attrape” les éléments (getElementById)

Page 3 / 9

En JavaScript, tu as fait : document.getElementById("numeroa"). Ça veut dire : “Va dans la page, trouve l’élément qui a l’id numeroa, et donne-le moi.” 

const Num_A = document.getElementById("numeroa");
const Num_B = document.getElementById("numerob");

const suma = document.getElementById("suma");
const resta = document.getElementById("resta");
const multiplicacion = document.getElementById("multiplicacion");
const division = document.getElementById("division");

Image (niveau enfant)

Idée : l’id = une étiquette unique. JS la lit et trouve l’objet.

Image (niveau junior dev)

Bon réflexe : garder ces références en variables constantes.
Doc (FR) : Document.getElementById().

Étape 2 — Lire ce que l’utilisateur écrit (value) + parseInt

Page 4 / 9

Les cases <input> contiennent du texte (même si tu as mis type="number"). Donc Num_A.value et Num_B.value sont des chaînes (strings).

Pour faire un vrai calcul, tu as utilisé parseInt(...) pour transformer ce texte en nombre entier. 

parseInt(Num_A.value)
parseInt(Num_B.value)

Image (niveau enfant)

Comme lire une étiquette : tu vois “12” écrit, mais tu veux le vrai nombre 12 pour calculer.

Image (niveau junior dev)

Doc (FR) : parseInt().
Attention (important pour comprendre) : si l’utilisateur laisse vide, parseInt("") donne NaN (pas un nombre). Dans ton exercice actuel, tu ne gères pas ce cas, donc ça peut afficher un résultat “bizarre”. C’est normal pour un premier exercice.

Étape 3 — Calculer (+, −, ×, ÷)

Page 5 / 9

Tu fais 4 calculs différents. Chaque calcul est mis dans une variable let (par exemple res_suma). 

let res_suma = parseInt(Num_A.value) + parseInt(Num_B.value);
let res_resta = parseInt(Num_A.value) - parseInt(Num_B.value);
let res_multiplicacion = parseInt(Num_A.value) * parseInt(Num_B.value);
let res_division = parseInt(Num_A.value) / parseInt(Num_B.value);

Image (niveau enfant)

Tu prends les mêmes 2 nombres et tu fais 4 actions.

Image (niveau junior dev)

En JavaScript, diviser par 0 ne “crash” pas : ça peut donner Infinity (ou NaN selon le cas). Doc : Infinity.

Étape 4 — Afficher sur la page (innerHTML + texte dynamique)

Page 6 / 9

Pour afficher le résultat, tu modifies le contenu des paragraphes. Tu as utilisé innerHTML. 

suma.innerHTML = `la suma de ${Num_A.value} + ${Num_B.value} es ${res_suma}`;
resta.innerHTML = `la resta de ${Num_A.value} - ${Num_B.value} es ${res_resta}`;
multiplicacion.innerHTML = `la multiplicacion de ${Num_A.value} * ${Num_B.value} es ${res_multiplicacion}`;
division.innerHTML = `la division de ${Num_A.value} / ${Num_B.value} es ${res_division}`;

Les textes entre ` ... ` s’appellent des template literals. Les morceaux ${...} sont remplacés par les valeurs.

Image (niveau enfant)

Idée : le paragraphe est un “endroit vide” où JS peut écrire.

Image (niveau junior dev)

Docs (FR) : Template literals et innerHTML.

Erreurs fréquentes + résumé à mémoriser

Page 7 / 9

Erreurs fréquentes (FR)

- Oublier un id dans le HTML → getElementById renvoie null.
- Laisser une case vide → parseInt("") → NaN.
- Diviser par 0 → résultat spécial (Infinity).
- Écrire + sans convertir → parfois “coller du texte” au lieu d’additionner.

Resumen (ES)

- Si falta un id → getElementById puede devolver null.
- Si el input está vacío → parseInt("") → NaN.
- División por 0 → puede salir Infinity.
- Sin convertir a número, el + puede concatenar.

Le résumé “guide” (FR) pour refaire l’exercice :

1) Je donne des id aux inputs et aux paragraphes → 2) Je les récupère en JS → 3) Je lis .value → 4) Je convertis avec parseInt → 5) Je calcule → 6) J’affiche avec innerHTML.

Bonus (optionnel, pour plus tard) : tu peux améliorer ton exercice en vérifiant que les valeurs sont bien des nombres avant de calculer (ex : si c’est vide, afficher un message d’erreur).

1) FR : À quoi sert un id en HTML ? / ES : ¿Para qué sirve un id en HTML?

FR : Un id sert à donner un nom unique à un élément pour que JavaScript le retrouve facilement.
Pourquoi : ton code utilise document.getElementById("..."), donc il a besoin d’un id exact.

ES : Un id da un nombre único a un elemento para que JavaScript lo encuentre fácil.
Por qué : tu código usa document.getElementById("..."), necesita ese id.

2) FR : Que fait document.getElementById(...) ? / ES : ¿Qué hace document.getElementById(...)?

FR : Ça retourne l’élément de la page qui a cet id (ou null s’il n’existe pas).
Pourquoi : tu stockes ces éléments dans Num_A, suma, etc.

ES : Devuelve el elemento con ese id (o null si no existe).
Por qué : lo guardas en variables como Num_A, suma, etc.

3) FR : Que contient Num_A.value ? / ES : ¿Qué contiene Num_A.value?

FR : Le texte que l’utilisateur a tapé dans l’input #numeroa (souvent une string).
Pourquoi : c’est cette valeur que tu convertis puis calcules.

ES : El texto que el usuario escribió en el input #numeroa (normalmente string).
Por qué : esa es la entrada para convertir y calcular.

4) FR : Pourquoi utiliser parseInt ici ? / ES : ¿Por qué usar parseInt aquí?

FR : Pour transformer le texte en nombre entier, sinon on risque de faire des opérations “texte”.
Pourquoi : en JS, "2" + "3" donne "23" (concaténation).

ES : Para convertir texto a número entero, si no puedes hacer operaciones como texto.
Por qué : en JS, "2" + "3" da "23".

5) FR : Quand la fonction calcular() s’exécute ? / ES : ¿Cuándo se ejecuta calcular()?

FR : Quand tu cliques sur le bouton “Aritmética”, via onclick="calcular()".
Pourquoi : l’attribut onclick appelle la fonction au clic.

ES : Cuando haces clic en el botón, por onclick="calcular()".
Por qué : onclick dispara la función al hacer clic.

6) FR : Pourquoi les paragraphes ont un id ? / ES : ¿Por qué los párrafos tienen id?

FR : Pour que JavaScript puisse écrire le résultat exactement au bon endroit.
Pourquoi : tu fais suma.innerHTML = ... et il faut donc l’élément #suma.

ES : Para que JavaScript escriba el resultado en el lugar correcto.
Por qué : haces suma.innerHTML = ..., necesitas #suma.

7) FR : À quoi sert innerHTML dans ton exercice ? / ES : ¿Para qué sirve innerHTML?

FR : À remplacer le contenu du paragraphe par une phrase avec le résultat.
Pourquoi : c’est comme “écrire” dans la page après le calcul.

ES : Para reemplazar el contenido del párrafo con una frase y el resultado.
Por qué : es “escribir” en la página después del cálculo.

8) FR : Pourquoi utiliser des backticks `...` ? / ES : ¿Por qué usar `...` (backticks)?

FR : Pour écrire une phrase et mettre des valeurs à l’intérieur avec ${...}.
Pourquoi : c’est plus lisible que faire beaucoup de "..." + variable + "...".

ES : Para escribir una frase e insertar valores con ${...}.
Por qué : es más legible que concatenar con +.

9) FR : Pourquoi tu as créé res_suma, res_resta, etc. ? / ES : ¿Por qué creaste res_suma, res_resta...?

FR : Pour stocker chaque résultat et l’utiliser ensuite dans le texte affiché.
Pourquoi : ça rend le code plus clair, tu sais quel résultat correspond à quoi.

ES : Para guardar cada resultado y usarlo en el texto.
Por qué : mejora la claridad del código.

10) FR : Si Num_A.value est vide, que se passe-t-il ? / ES : Si Num_A.value está vacío, ¿qué pasa?

FR : parseInt("") donne NaN, donc les calculs donnent NaN aussi.
Pourquoi : NaN signifie “pas un nombre”, et un calcul avec NaN reste NaN.

ES : parseInt("") da NaN, y los cálculos salen NaN.
Por qué : NaN significa “no es un número”.

11) FR : Que veut dire NaN ? / ES : ¿Qué significa NaN?

FR : “Not a Number” = ce n’est pas un nombre valide.
Pourquoi : JavaScript l’utilise quand une conversion en nombre échoue.

ES : “Not a Number” = no es un número válido.
Por qué : aparece cuando falla una conversión a número.

12) FR : Pourquoi le script app.js est à la fin du body ? / ES : ¿Por qué app.js está al final del body?

FR : Pour que le HTML soit déjà chargé quand JS cherche les éléments par id.
Pourquoi : sinon getElementById peut ne rien trouver au moment où le script s’exécute.

ES : Para que el HTML ya esté cargado cuando JS busca los elementos por id.
Por qué : si no, getElementById puede devolver null.

13) FR : À quoi sert type="number" sur input ? / ES : ¿Para qué sirve type="number"?

FR : Ça aide le navigateur à proposer un clavier numérique et à limiter certains caractères.
Pourquoi : ça améliore l’expérience utilisateur, mais .value reste du texte.

ES : Ayuda al navegador con teclado numérico y algunas validaciones.
Por qué : mejora UX, pero .value sigue siendo texto.

14) FR : Pourquoi utiliser const pour Num_A et suma ? / ES : ¿Por qué usar const?

FR : Parce que tu ne changes pas la référence de l’élément, tu l’utilises juste.
Pourquoi : const évite de réassigner par erreur.

ES : Porque no cambias la referencia del elemento, solo la usas.
Por qué : const evita reasignaciones por error.

15) FR : Pourquoi tu as utilisé let pour res_suma ? / ES : ¿Por qué let para res_suma?

FR : Tu peux stocker une valeur calculée dans une variable locale. Ici elle ne change pas après, mais c’est OK.
Pourquoi : let permet une variable modifiable ; dans ce cas const aurait aussi marché.

ES : Guardas un resultado en una variable local. Aquí no cambia luego, pero está bien.
Por qué : let permite reasignar; aquí const también serviría.

16) FR : Quelle partie “relie” HTML et JS ? / ES : ¿Qué parte “conecta” HTML y JS?

FR : Les id + getElementById (pour trouver), et onclick (pour lancer).
Pourquoi : sans ces liens, JS ne sait pas où lire/écrire, ni quand calculer.

ES : Los id + getElementById (encontrar), y onclick (ejecutar).
Por qué : sin eso, JS no sabe dónde leer/escribir ni cuándo calcular.

17) FR : Quel est le rôle du bouton ? / ES : ¿Cuál es el papel del botón?

FR : Il déclenche l’action : appeler calcular() au bon moment.
Pourquoi : ça évite que ça calcule tout le temps pendant que tu écris.

ES : Dispara la acción: llamar a calcular().
Por qué : evita calcular todo el tiempo mientras escribes.

18) FR : Pourquoi on écrit dans 4 paragraphes différents ? / ES : ¿Por qué 4 párrafos?

FR : Parce que tu affiches 4 résultats séparés, un endroit par résultat.
Pourquoi : c’est plus clair pour l’utilisateur (et pour le code).

ES : Porque muestras 4 resultados separados, un sitio por resultado.
Por qué : es más claro para el usuario y el código.

19) FR : Quelle est la différence entre valeur entrée et résultat ? / ES : Diferencia entre entrada y resultado

FR : Les entrées viennent des inputs (Num_A.value), le résultat vient du calcul (res_suma).
Pourquoi : tu affiches les deux : “tu as écrit X et Y, le résultat est Z”.

ES : La entrada viene del input (Num_A.value), el resultado del cálculo (res_suma).
Por qué : muestras “X e Y → Z”.

20) FR : Si tu devais refaire l’exercice, quelles sont les 6 étapes ? / ES : ¿Cuáles son los 6 pasos?

FR : (1) Mettre des id → (2) récupérer avec getElementById → (3) lire .value → (4) convertir (parseInt) → (5) calculer → (6) afficher (innerHTML).
Pourquoi : c’est exactement l’ordre logique de ton code.

ES : (1) Poner id → (2) getElementById → (3) .value → (4) convertir (parseInt) → (5) calcular → (6) mostrar (innerHTML).
Por qué : es el orden lógico de tu ejercicio.

<script>
  document.addEventListener("DOMContentLoaded", () => {
    const stopSpeech = () => {
      if (!("speechSynthesis" in window)) return;
      window.speechSynthesis.cancel();
    };

    const normalizeSpeakableText = (rawText) =>
      String(rawText || "")
        .replace(/\s+\n/g, "\n")
        .replace(/\n{3,}/g, "\n\n")
        .replace(/[ \t]{2,}/g, " ")
        .trim();

    const getSpeakableTextFromElement = (element) => {
      if (!element) return "";
      const cloned = element.cloneNode(true);
      cloned.querySelectorAll("pre, code, script, style, .nav-buttons").forEach((el) => el.remove());
      return normalizeSpeakableText(cloned.innerText || "");
    };

    const getPreferredVoice = (lang) => {
      const availableVoices = window.speechSynthesis.getVoices?.() || [];
      return (
        availableVoices.find((voice) => voice.lang?.toLowerCase() === lang.toLowerCase()) ||
        availableVoices.find((voice) => voice.lang?.toLowerCase().startsWith(lang.toLowerCase().split("-")[0])) ||
        null
      );
    };

    const speakTextChunks = (textChunks, lang) => {
      if (!("speechSynthesis" in window)) return;
      const chunks = (Array.isArray(textChunks) ? textChunks : []).map(normalizeSpeakableText).filter(Boolean);
      if (!chunks.length) return;

      stopSpeech();

      const preferredVoice = getPreferredVoice(lang);
      let currentIndex = 0;

      const speakNext = () => {
        const nextText = chunks[currentIndex];
        if (!nextText) return;

        const utterance = new SpeechSynthesisUtterance(nextText);
        utterance.lang = lang;
        if (preferredVoice) utterance.voice = preferredVoice;

        utterance.onend = () => {
          currentIndex += 1;
          if (currentIndex < chunks.length) speakNext();
        };

        window.speechSynthesis.speak(utterance);
      };

      speakNext();
    };

    const speakSection = (sectionElement, lang) => {
      const sectionText = getSpeakableTextFromElement(sectionElement);
      speakTextChunks([sectionText], lang);
    };

    const speakWholeGuide = (lang) => {
      const titleElement = document.querySelector("header .title");
      const titleText = getSpeakableTextFromElement(titleElement);

      const sectionTexts = Array.from(document.querySelectorAll("section.page[id]")).map((sectionElement) =>
        getSpeakableTextFromElement(sectionElement)
      );

      speakTextChunks([titleText, ...sectionTexts], lang);
    };

    const addTtsControls = () => {
      if (!("speechSynthesis" in window)) return;

      const titleContainer = document.querySelector("header .title");
      if (titleContainer && !titleContainer.querySelector(".tts-controls--global")) {
        const globalControls = document.createElement("div");
        globalControls.className = "tts-controls tts-controls--global";
        globalControls.setAttribute("aria-label", "Audio (lecture complète)");

        const allFrBtn = document.createElement("button");
        allFrBtn.type = "button";
        allFrBtn.className = "tts-btn";
        allFrBtn.textContent = "🔊 Tout FR";
        allFrBtn.addEventListener("click", (event) => {
          event.preventDefault();
          event.stopPropagation();
          speakWholeGuide("fr-FR");
        });

        const allEsBtn = document.createElement("button");
        allEsBtn.type = "button";

        allEsBtn.addEventListener("click", (event) => {
          event.preventDefault();
          event.stopPropagation();
          speakWholeGuide("es-ES");
        });

        const stopAllBtn = document.createElement("button");
        stopAllBtn.type = "button";
        stopAllBtn.className = "tts-btn tts-btn--stop";
        stopAllBtn.textContent = "⏹ Stop";
        stopAllBtn.addEventListener("click", (event) => {
          event.preventDefault();
          event.stopPropagation();
          stopSpeech();
        });

        globalControls.append(allFrBtn, allEsBtn, stopAllBtn);
        titleContainer.append(globalControls);
      }

      document.querySelectorAll("section.page[id]").forEach((sectionElement) => {
        const header = sectionElement.querySelector(".page-header");
        if (!header) return;
        if (header.querySelector(".tts-controls")) return;

        const controls = document.createElement("div");
        controls.className = "tts-controls";
        controls.setAttribute("aria-label", "Audio (lecture)");

        const frBtn = document.createElement("button");
        frBtn.type = "button";
        frBtn.className = "tts-btn";
        frBtn.textContent = "🔊 FR";
        frBtn.addEventListener("click", (event) => {
          event.preventDefault();
          event.stopPropagation();
          speakSection(sectionElement, "fr-FR");
        });

        const esBtn = document.createElement("button");
        esBtn.type = "button";
        esBtn.className = "tts-btn";
        esBtn.textContent = "🔊 ES";
        esBtn.addEventListener("click", (event) => {
          event.preventDefault();
          event.stopPropagation();
          speakSection(sectionElement, "es-ES");
        });

        const stopBtn = document.createElement("button");
        stopBtn.type = "button";
        stopBtn.className = "tts-btn tts-btn--stop";
        stopBtn.textContent = "⏹ Stop";
        stopBtn.addEventListener("click", (event) => {
          event.preventDefault();
          event.stopPropagation();
          stopSpeech();
        });

        controls.append(frBtn, esBtn, stopBtn);
        header.append(controls);

        header.style.cursor = "pointer";
        header.addEventListener("click", () => {
          speakSection(sectionElement, "fr-FR");
        });
      });
    };

    window.speechSynthesis.onvoiceschanged = addTtsControls;
    addTtsControls();
  });
