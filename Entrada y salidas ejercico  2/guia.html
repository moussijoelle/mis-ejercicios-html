Guía del ejercicio: Promedio de notas

1. Resumen de lo que hemos hecho en el proyecto

  • La página tiene un título de web en el <head> y un <h1> visible con el mismo nombre: Promedio de notas. No usamos la palabra «Ejercicio» porque el h1 es el título de la página para el usuario, no el número de tarea del curso.
  • Un párrafo <p> explica el enunciado: pedir 3 notas por formulario, calcular el promedio y usar parseFloat.
  • El formulario (labels e inputs) está fuera de <details> y <summary>. Esos elementos sirven para secciones plegables; el formulario debe verse siempre, no escondido dentro del summary.
  • El JavaScript en app.js escucha el envío del formulario, convierte las notas a número, calcula el promedio y lo muestra en un párrafo con id="resultado".

2. Enunciado del ejercicio

Haz un programa que le pida al usuario que introduzca 3 notas por formulario y que muestre como resultado el promedio de las 3 notas por pantalla. Usa la función parseFloat para convertir a número.

4. Explicación línea a línea del JavaScript (app.js)

1 const formularioNotas = document.getElementById("form-notas"); — Guarda la referencia al formulario del HTML.

2 const nota1 = document.getElementById("nota1"); — Referencia al primer input.

3 const nota2 = document.getElementById("nota2"); — Referencia al segundo input.

4 const nota3 = document.getElementById("nota3"); — Referencia al tercer input.

5 const resultado = document.getElementById("resultado"); — Referencia al párrafo donde se mostrará el promedio.

7 formularioNotas.addEventListener("submit", (evento) => { — Cuando el usuario envía el formulario, ejecuta esta función.

8 evento.preventDefault(); — Evita que la página se recargue (comportamiento por defecto del submit).

10 const n1 = parseFloat(nota1.value); — Lee el texto del input y lo convierte a número decimal.

11 const n2 = parseFloat(nota2.value); — Igual con la segunda nota.

12 const n3 = parseFloat(nota3.value); — Igual con la tercera nota.

14 const promedio = (n1 + n2 + n3) / 3; — Suma las tres notas y divide entre 3.

16 resultado.textContent = `Promedio: ${promedio}`; — Escribe el resultado en el párrafo del HTML.

17 }); — Cierra la función del listener y la llamada a addEventListener.

5. Test: 15 preguntas (abre cada respuesta solo cuando la necesites)

1. ¿Para qué sirve el elemento <title> en el head?

Define el texto que aparece en la pestaña del navegador y en los marcadores. No es el título grande visible en la página; eso es el <h1>.

2. ¿Cuál es la diferencia entre <title> y <h1>?

<title> va en el <head> y solo se ve en la pestaña. <h1> va en el <body> y es el título principal que el usuario lee en la página.

3. ¿Por qué usamos for en el label y id en el input?

Conectan label e input: al pulsar el texto del label, el cursor va al campo correcto. Mejora la accesibilidad y la usabilidad.

4. ¿Qué hace el atributo required en un input?

Obliga a rellenar el campo antes de enviar el formulario. El navegador muestra un aviso si está vacío.

5. ¿Qué significa type="number" y step="0.01"?

type="number" indica que el valor debe ser numérico. step="0.01" permite decimales con dos cifras después de la coma (por ejemplo 7.5).

6. ¿Para qué sirve id="form-notas" en el formulario?

Permite seleccionar ese formulario en JavaScript con document.getElementById("form-notas") y escuchar su envío.

7. ¿Qué hace document.getElementById?

Busca en el HTML un elemento que tenga el id indicado y devuelve ese elemento para usarlo en JavaScript.

8. ¿Qué es el evento submit?

Se dispara cuando el usuario envía el formulario (botón «Calcular promedio» o Enter en un campo). JavaScript puede reaccionar con addEventListener("submit", ...).

9. ¿Por qué usamos evento.preventDefault()?

El envío normal del formulario recarga la página. preventDefault() cancela eso para que solo actualicemos el texto del resultado con JavaScript.

10. ¿Qué hace parseFloat?

Convierte un texto (por ejemplo "8.5") en número decimal. Los valores de input.value siempre llegan como cadena de texto.

11. ¿Cuál es la fórmula del promedio de tres notas?

(nota1 + nota2 + nota3) / 3. Se suman las tres notas y se divide el total entre 3.

12. ¿Qué diferencia hay entre textContent e innerHTML?

textContent escribe solo texto plano (más seguro). innerHTML interpreta etiquetas HTML; aquí usamos textContent porque solo mostramos un número.

13. ¿Por qué el formulario no debe ir dentro de <summary>?

<summary> es solo la línea clicable que abre/cierra un <details>. Labels e inputs deben estar fuera, en el cuerpo visible del formulario.

14. Si las notas son 6, 8 y 10, ¿qué promedio debe mostrar el programa?

(6 + 8 + 10) / 3 = 8. En pantalla debería verse algo como: Promedio: 8.

15. ¿Qué pasa si dejamos un input vacío y quitamos required?

parseFloat("") devuelve NaN (Not a Number). El promedio también sería NaN. Por eso conviene required o validar en JavaScript antes de calcular.
