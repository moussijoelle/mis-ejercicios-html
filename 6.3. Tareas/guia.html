Guía del ejercicio: Lista de tareas

Objetivo: crear una pequeña app que guarde tareas y tenga un menú con 4 botones.

El enunciado pide:

  1. Añadir tarea: guardar una tarea nueva.
  2. Mostrar todas las tareas: enseñar el listado completo.
  3. Eliminar tarea: borrar una tarea del listado.
  4. Salir: mostrar la palabra Gracias en pantalla y deshabilitar los botones (como en “Galletitas”).

Estructura del proyecto

  1. index.html: contiene la interfaz (botones, input, lista, mensaje).
  2. js/app.js: contiene la lógica (guardar tareas, pintar lista, eliminar, salir).
  3. guia.html: este archivo (explicación y quiz).

Cómo funciona (en 15 segundos)

  1. Las tareas se guardan en un array: tareasPendientes.
  2. Cuando añades/elimininas, se vuelve a “pintar” la lista en el <ol id="lista-tareas">.
  3. Al salir, se escribe Gracias y se deshabilitan los botones (y el input).

Explicación por líneas (HTML)

  1. <!DOCTYPE html>: indica que el documento es HTML moderno.
  2. <html lang="es">: idioma español (ayuda a navegadores y accesibilidad).
  3. <head>: configuración de la página (no se ve en el contenido).
  4. <meta charset="UTF-8">: permite caracteres como ñ y tildes.
  5. <meta name="viewport" ...>: hace que se vea bien en móviles.
  6. <script src="...tailwind...">: carga Tailwind en modo navegador (para usar clases).
  7. <style type="text/tailwindcss">: estilos usando Tailwind con @apply.
  8. <body ...>: empieza el contenido visible de la página.
  9. <div class="grid ... md:grid-cols-3">: crea un layout con 1 columna en móvil y 3 columnas desde md.
  10. Columna 1: <h1> (título), <input id="texto-nueva-tarea"> (escribir tarea), <ol id="lista-tareas"> (donde se pintan tareas) y <p id="mensaje"> (mensajes).
  11. Columna 2: 4 botones con id específicos. El JavaScript los necesita para encontrarles y añadirles eventos.
  12. Columna 3: imagen/gif decorativo (no es obligatorio para el ejercicio).
  13. <script src="js/app.js">: carga el JavaScript para que los botones funcionen.

Tu JavaScript (js/app.js) explicado línea por línea

Esta es la lógica que hace que la app funcione.

const tareasPendientes = [];

Explicación por líneas (JS)

  1. const tareasPendientes = [];: array donde se guardan todas las tareas.
  2. document.getElementById("..."): busca elementos en el HTML por su id (input, lista, mensaje y botones).
  3. const botonesMenu = [...]: una lista con los 4 botones para poder deshabilitarlos en “Salir”.
  4. setMensaje(texto): muestra un mensaje en #mensaje. Si el texto está vacío, pone un espacio para que no “salte” el layout.
  5. actualizarListado(): borra la lista y crea un <li> por cada tarea del array.
  6. anadirTarea(): lee el input, valida que no esté vacío, añade al array y actualiza la lista.
  7. mostrarTodasLasTareas(): pinta la lista y si está vacía muestra “No hay tareas.”.
  8. eliminarTarea(): pide un número por prompt, valida, hace splice y repinta.
  9. deshabilitarMenu(): pone disabled = true a botones (y al input).
  10. salirDeLaAplicacion(): escribe “Gracias” y deshabilita el menú.
  11. addEventListener("click", ...): conecta cada botón con su función.
  12. keydown en el input: si pulsas Enter, añade tarea (comodidad extra).

Quiz (15 preguntas tipo ABC)

Haz el quiz. Las respuestas están ocultas (abre cada <details> al final para comprobar).

1) ¿Qué guarda el array tareasPendientes?

A) Botones del menú    B) Tareas por hacer    C) Imágenes

Ver respuesta

B. Guarda los textos de las tareas.

2) ¿Qué etiqueta es buena para mostrar una lista numerada?

A) <ol>    B) <div>    C) <span>

Ver respuesta

A. <ol> es una lista ordenada (numerada).

3) ¿Qué hace document.getElementById("mensaje")?

A) Crea un botón    B) Busca un elemento por id    C) Borra el HTML

Ver respuesta

B. Obtiene el elemento con ese id.

4) ¿Qué botón debe deshabilitar todos los botones?

A) Añadir    B) Mostrar    C) Salir

Ver respuesta

C. “Salir de la aplicación”.

5) ¿Qué se muestra al pulsar “Salir”?

A) Hola    B) Gracias    C) Adiós

Ver respuesta

B. Debe aparecer la palabra “Gracias”.

6) ¿Qué hace innerHTML = "" en la lista?

A) Vacía la lista    B) Cambia el color    C) Añade un elemento

Ver respuesta

A. Borra el contenido antes de repintar.

7) ¿Para qué sirve addEventListener("click", ...)?

A) Para reaccionar al clic    B) Para guardar tareas    C) Para cambiar el título

Ver respuesta

A. Conecta un botón con una función cuando haces clic.

8) ¿Qué devuelve prompt(...) si el usuario cancela?

A) 0    B) null    C) true

Ver respuesta

B. Devuelve null.

9) Si el usuario escribe una tarea con espacios al inicio y al final, ¿qué se usa para limpiarlo?

A) trim()    B) push()    C) splice()

Ver respuesta

A. trim() quita espacios al principio y al final.

10) ¿Qué hace push en un array?

11) ¿Qué hace splice(indice, 1)?

B. Borra una tarea del array.
