1. Fichiers du projet et rôle de chacun

Ton mini-projet repose sur trois éléments principaux :

  • index.html — C'est le document que le navigateur charge en premier. Il décrit la structure de la page (titres, texte, images, bouton) et indique où trouver les styles (Tailwind via CDN) et le script (app.js). Le HTML ne « calcule » pas le hasard : il se contente d'afficher des éléments et de donner des identifiants (id) pour que le JavaScript puisse les retrouver.
  • app.js — C'est le fichier de comportement. Il contient la fonction tirarDado qui tire un nombre entre 1 et 6, construit le chemin du fichier image, et met à jour l'attribut src de l'image du résultat. Il contient aussi le code qui enregistre l'écoute du clic sur le bouton une fois la page prête.
  • Dossier img/ — Il contient les six images dado1.png à dado6.png. Le navigateur les charge lorsque l'URL du src pointe vers ces fichiers (par exemple img/dado4.png). Si un chemin est faux ou si un fichier manque, l'image ne s'affiche pas correctement.

2. Vue d'ensemble du fonctionnement

Quand tu ouvres index.html, le navigateur construit une représentation interne de la page appelée le DOM (Document Object Model). Chaque balise devient un nœud : par exemple ton bouton et ton image résultat sont des nœuds auxquels le JavaScript peut accéder par leur id.

Ensuite le navigateur charge et exécute app.js. Le bloc DOMContentLoaded attend que le HTML soit prêt, puis attache un écouteur d'événement sur le bouton : « à chaque clic, appelle tirarDado ». Quand tu cliques, tirarDado calcule un entier aléatoire entre 1 et 6, fabrique une chaîne de caractères qui correspond au chemin du fichier PNG (concaténation), et assigne cette chaîne à document.getElementById("imagenDado").src. Le navigateur voit que le src a changé et charge l'image correspondante : tu vois une nouvelle face du dé.

3. HTML (index.html) : explications détaillées

3.1 En-tête du document (<head>)

<!DOCTYPE html> indique au navigateur qu'il s'agit d'un document HTML5 moderne. <html lang="es"> enveloppe toute la page ; l'attribut lang aide l'accessibilité et les moteurs (ici l'énoncé est en espagnol).

<meta charset="UTF-8"> garantit que les accents et caractères spéciaux s'affichent correctement. viewport aide au rendu sur mobile.

La balise <script src="…@tailwindcss/browser@4"> charge Tailwind dans le navigateur pour que les classes utilitaires (class="…") et le bloc <style type="text/tailwindcss"> puissent styliser la page sans fichier CSS séparé.

Le bloc @layer base définit des styles par défaut pour p, button, details, summary, img, etc. Par exemple, les images ont une largeur fixe de 150px pour un affichage homogène des faces du dé.

<link rel="shortcut icon" …> demande une petite icône d'onglet (si le fichier favicon.ico existe). <title> donne le titre de l'onglet du navigateur.

3.2 Corps (<body>) et mise en page

class="container w-5/6 m-auto" centre le contenu et limite sa largeur grâce à Tailwind.

Le <h2> affiche le titre « Ejercicios ». L'attribut id="mensaje1" pourrait servir plus tard en JavaScript si tu voulais modifier ce titre dynamiquement (ce n'est pas utilisé dans ton app.js actuel).

3.3 Bloc repliable <details> / <summary>

Ce motif HTML permet d'afficher un titre cliquable (« 1. Practicar número random ») et de replier ou déplier le reste du contenu. C'est pratique pour un polycopié d'exercices : l'utilisateur ouvre la section qui l'intéresse.

3.4 Énoncé et piste

Les paragraphes <p> contiennent l'énoncé en espagnol et la piste : changer le src de l'image en concaténant du texte avec le nombre aléatoire. Ta version utilise le préfixe img/ parce que les fichiers PNG sont dans ce dossier.

3.5 Les six images « Descargar dados »

Ces six balises <img> servent de aperçu des fichiers disponibles. Elles ne sont pas modifiées par ton script : elles montrent simplement à l'utilisateur à quoi ressemblent les faces 1 à 6.

3.6 L'image résultat id="imagenDado"

C'est l'image unique dont le src sera mis à jour par JavaScript. L'attribut id doit être unique dans la page : ainsi getElementById("imagenDado") cible exactement ce nœud. Le src initial img/dado1.png est la face affichée au chargement avant le premier clic.

3.7 Le bouton id="btnLanzarDado"

type="button" évite tout comportement de soumission de formulaire si la page était un jour placée dans un <form>. Il n'y a pas d'attribut onclick dans le HTML : le clic est géré dans app.js avec addEventListener, ce qui sépare mieux structure et comportement.

3.8 Chargement du script app.js

La balise <script src="app.js"></script> placée à la fin du body garantit que le DOM (bouton, image, etc.) existe déjà lorsque le fichier est exécuté. Même avec DOMContentLoaded, cette position reste une bonne habitude pour des pages simples.

Tu peux copier-coller ce texte dans un fichier nommé index.html à la racine du projet (à côté de app.js et du dossier img).

5. JavaScript (app.js) : explications détaillées

5.1 La fonction tirarDado

Une fonction regroupe des instructions réutilisables. Ici, elle encapsule tout ce qui doit se passer quand l'utilisateur « lance le dé ».

const numeroRandom = Math.floor(Math.random() * 6) + 1; — Math.random() renvoie un nombre décimal pseudo-aléatoire dans l'intervalle [0, 1[ (zéro inclus, un exclu). Le multiplier par 6 donne [0, 6[. Math.floor tronque vers le bas : on obtient un entier parmi 0, 1, 2, 3, 4, 5. Ajouter 1 décale l'intervalle vers 1…6, ce qui correspond aux faces d'un dé classique.

const rutaImagen = "img/dado" + numeroRandom + ".png"; — En JavaScript, l'opérateur + entre une chaîne et un nombre convertit le nombre en texte et concatène. Ainsi, si numeroRandom vaut 4, la variable contient la chaîne img/dado4.png, qui est l'URL relative correcte vers ton fichier dans le dossier img.

document.getElementById("imagenDado").src = rutaImagen; — document représente la page chargée. getElementById renvoie l'élément DOM correspondant. La propriété src d'une image est l'adresse de la ressource affichée ; lui assigner une nouvelle chaîne demande au navigateur de charger la nouvelle image (souvent avec mise en cache si le fichier a déjà été vu).

5.2 L'écouteur DOMContentLoaded

document.addEventListener("DOMContentLoaded", () => { … }); enregistre une fonction (ici une fonction fléchée) qui s'exécute une fois le document HTML analysé et le DOM construit. Cela évite d'appeler getElementById("btnLanzarDado") trop tôt, avant que le bouton n'existe.

5.3 L'écouteur de clic sur le bouton

.addEventListener("click", tirarDado) attache la fonction tirarDado comme gestionnaire du clic. On écrit tirarDado sans parenthèses pour passer la référence de la fonction (le navigateur l'invoquera au moment du clic). Écrire tirarDado() ici appellerait la fonction immédiatement au chargement, ce qui n'est pas souhaité.

7. Quinze questions / réponses (approfondies)

Pourquoi parle-t-on de JavaScript et pas de « Java » ?
Java et JavaScript sont deux langages différents, malgré le nom proche. Ici, tout se passe dans le navigateur avec un fichier .js : c'est du JavaScript. Java sert souvent à d'autres types d'applications (serveurs, Android, etc.).
Quelle est la différence entre HTML et JavaScript dans ce devoir ?
Le HTML décrit ce qui est affiché et la structure (titres, textes, images, bouton). Le JavaScript décrit ce qui change quand l'utilisateur agit : calcul d'un nombre et mise à jour de l'image. Sans JS, la page resterait statique après chargement.
Pourquoi une seule image pour le résultat au lieu de six ?
Une seule balise <img> suffit pour montrer une face à la fois. On simule le changement de face en modifiant son src. C'est plus simple à maintenir et c'est exactement ce que demande la piste de l'exercice (concaténation + src).
Pourquoi utiliser un id sur l'image et sur le bouton ?
Les id doivent être uniques dans la page. Ils offrent un moyen stable et rapide de retrouver un élément précis depuis JavaScript avec getElementById. Sans id, il faudrait d'autres sélecteurs (classes, position dans le DOM), plus fragiles pour un petit exercice.
Pourquoi le chemin contient img/ ?
Parce que tes fichiers PNG sont rangés dans le dossier img. Le navigateur résout les URL relatives par rapport à l'emplacement de index.html. Si tu mettais seulement dado3.png alors que le fichier est dans img/, le navigateur chercherait au mauvais endroit.
Que se passerait-il si app.js était chargé dans le <head> sans defer ?
L'exécution bloquerait le rendu jusqu'à la fin du script, et selon l'ordre du code, getElementById pourrait s'exécuter avant que les balises du body n'existent — d'où l'intérêt soit de placer le script en bas du body, soit d'utiliser defer, soit d'attendre DOMContentLoaded comme tu le fais pour l'enregistrement du clic.
Pourquoi éviter onclick="tirarDado()" dans le HTML ?
Les gestionnaires inline mélangent structure et logique. Les placer dans app.js améliore la lisibilité, facilite les politiques de sécurité (CSP) et regroupe le comportement au même endroit.
Comment déboguer si « rien ne se passe » ?
Ouvre les outils développeur (F12), onglet Console : une erreur JavaScript (par exemple élément introuvable, chemin d'image incorrect) y apparaît souvent en rouge. Vérifie aussi l'onglet Réseau pour voir si les requêtes vers img/dadoX.png échouent (404).
Le hasard est-il « vraiment » aléatoire ?
Math.random() produit des nombres pseudo-aléatoires, suffisants pour un dé pédagogique. Pour la cryptographie ou des tirages très sensibles, on utiliserait d'autres API plus spécialisées.
Que livrer pour le devoir ?
En général : index.html, app.js, et le dossier img avec les six PNG. Ce fichier resume-devoir-de.html est une aide personnelle à la compréhension, pas toujours demandé par l'enseignant.
Pourquoi utilise-t-on const pour numeroRandom et rutaImagen ?
const déclare une variable dont on ne reassignera pas la référence dans la suite de la fonction. Ici, chaque clic recrée une nouvelle exécution de tirarDado : à l'intérieur d'un appel, le nombre et le chemin ne changent pas après leur calcul. Cela rend l'intention claire et évite des réassignations accidentelles. (let servirait si tu devais modifier la variable plus tard dans le même bloc.)
À quoi sert l'attribut alt sur les <img> ?
Le texte alt décrit l'image pour les personnes qui utilisent un lecteur d'écran et s'affiche si l'image ne charge pas. Ce n'est pas utilisé par ton script du dé, mais c'est une bonne pratique d'accessibilité et de robustesse.
Pourquoi les images ont-elles une largeur de 150px dans le CSS du index.html ?
Dans la feuille Tailwind embarquée, la règle img { width: 150px; } impose une taille uniforme aux vignettes et à l'image résultat, pour que la page reste lisible et que les faces du dé aient toutes la même taille visuelle.
Qu'est-ce que le DOM concrètement ?
Quand le navigateur lit ton HTML, il construit une arborescence d'objets en mémoire : chaque balise devient un nœud avec des propriétés (src pour une image, etc.). Cette arborescence est le DOM. JavaScript (document, getElementById) manipule ces objets : changer .src sur le nœud image met à jour ce que l'utilisateur voit.
Peut-on appeler tirarDado sans cliquer (par exemple au chargement de la page) ?
Oui : tu pourrais ajouter tirarDado(); une fois après sa définition pour lancer un premier tirage automatique. Dans ton devoir actuel, seul le clic déclenche la fonction via addEventListener, ce qui correspond à l'idée d'un utilisateur qui « lance » le dé quand il le souhaite.

Note : ce fichier resume-devoir-de.html est une synthèse pédagogique. Ton projet fonctionnel reste index.html + app.js + img/.
