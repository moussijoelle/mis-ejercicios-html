1. En qué consiste el ejercicio

Dado un array de números, el programa debe mostrar:

  • La suma de todos los números pares.
  • La suma de todos los números impares.

Un número par es divisible por 2 sin resto (2, 4, 6, 8, 10…). Un número impar no lo es (1, 3, 5, 7, 9…).

Ejemplo: si introduces 2, 5, 8, 3, 10

  • Pares: 2 + 8 + 10 = 20
  • Impares: 5 + 3 = 8

Entrada: números escritos en un campo de texto, separados por comas.

Salida: dos totales mostrados en pantalla.

Texto: "2, 5, 8, 3, 10"
        ↓ split(',')
Array: ["2", " 5", " 8", " 3", " 10"]
        ↓ trim + Number
Números: [2, 5, 8, 3, 10]
        ↓ recorrer cada uno
¿Par?  → sumaPares
¿Impar? → sumaImpares
        ↓
Mostrar ambas sumas en #resultado

Ejemplo manual (como en clase)

Número¿Par o impar?Se suma en…
2Par (2 ÷ 2 = 1, resto 0)sumaPares
5ImparsumaImpares
8ParsumaPares
3ImparsumaImpares
10ParsumaPares

En este proyecto el HTML muestra la interfaz (input, botón, zona de resultados); la lógica está en js/app.js.

2. Explicación en francés (pour réviser)

L'exercice, en bref : tu as une liste de nombres. Le programme doit calculer deux choses : la somme de tous les nombres pairs et la somme de tous les nombres impairs.

Un nombre pair se divise par 2 sans reste (2, 4, 6…). Un nombre impair ne se divise pas par 2 (1, 3, 5…).

Exemple : 2, 5, 8, 3, 10 → pairs : 2+8+10 = 20 · impairs : 5+3 = 8.

L'utilisateur écrit les nombres dans une zone de texte, séparés par des virgules. Le programme utilise split(',') pour découper le texte en morceaux, comme on découpe une phrase en mots avec split(' ').

Étapes : lire le texte → découper en liste → convertir en vrais nombres → pour chaque nombre, tester pair ou impair → afficher les deux totaux.

3. El método split (analogía con frases)

Con una frase, el profesor suele enseñar esto:

let frase = "Hola mundo ejemplo";
const arrayPalabras = frase.split(' ');

split(' ') corta en cada espacio. Resultado: ["Hola", "mundo", "ejemplo"] — tres palabras.

En este ejercicio la idea es la misma, pero el separador es la coma:

let texto = "2, 5, 8, 3, 10";
const trozos = texto.split(',');

Resultado aproximado: ["2", " 5", " 8", " 3", " 10"]. Son cadenas de texto, no números todavía. Por eso en app.js usamos trim() (quita espacios) y Number() (convierte a número para poder sumar).

4. Pasos que sigue el programa

  1. La usuaria escribe números separados por comas y pulsa «Calcular sumas».
  2. JavaScript escucha el evento submit del formulario y evita que la página se recargue.
  3. Se lee el valor del input y se convierte en array con split(',').
  4. Cada trozo se limpia con trim() y se convierte con Number().
  5. Si algún valor no es un número válido, se muestra un mensaje de error.
  6. Se recorre el array: si numero % 2 === 0 es par; si no, impar.
  7. Se acumulan las sumas en sumaPares y sumaImpares.
  8. Se escribe el resultado en el div #resultado.

7. Test de 15 preguntas

Las respuestas están ocultas dentro de cada bloque <details>. Haz clic en cada pregunta solo cuando quieras ver la respuesta detallada. Así puedes hacer el test sin ver las soluciones de antemano.

1. ¿Qué debe calcular y mostrar este ejercicio?

Debe recibir una lista de números y mostrar dos resultados:

  1. La suma de todos los números pares del array.
  2. La suma de todos los números impares del array.

No pide mostrar la lista entera ni contar cuántos hay: solo las dos sumas finales.

2. ¿Qué es un número par y qué es un número impar?

Par: al dividirlo entre 2 el resto es 0. Ejemplos: 0, 2, 4, 8, 10.

Impar: al dividirlo entre 2 el resto no es 0. Ejemplos: 1, 3, 5, 7, 9.

En el código se comprueba con numero % 2 === 0: el operador % devuelve el resto de la división.

3. Con la entrada 2, 5, 8, 3, 10, ¿cuáles son las dos sumas correctas?

Pares (2, 8, 10): 2 + 8 + 10 = 20.

Impares (5, 3): 5 + 3 = 8.

El programa debe mostrar esos dos valores en #resultado después de pulsar «Calcular sumas».

4. ¿Por qué usamos split(',') y no split(' ') como en el ejemplo de frases?

En el ejemplo de frases el separador es el espacio entre palabras: "Hola mundo" → split(' ').

En este ejercicio el enunciado pide números separados por comas: "2, 5, 8". El carácter que marca dónde cortar es la coma, por eso split(',').

Si usáramos espacio con 2, 5, 8, no partiría bien porque la coma seguiría pegada al número (por ejemplo "2,").

5. ¿Para qué sirven trim() y Number() después del split?

Tras split(',') cada trozo es una cadena de texto, no un número. A menudo llevan espacios: " 5".

trim() elimina espacios al inicio y al final → "5".

Number() convierte la cadena en número real → 5, para poder usar += y %.

Sin Number(), JavaScript podría concatenar textos en lugar de sumar matemáticamente.

6. ¿Qué hace numero % 2 === 0 en el programa?

% es el operador módulo: devuelve el resto de dividir numero entre 2.

  • Si numero es 8: 8 % 2 = 0 → condición verdadera → es par.
  • Si numero es 5: 5 % 2 = 1 → condición falsa → va al else → es impar.

=== 0 comprueba de forma estricta que el resto sea exactamente cero.

7. ¿El cero (0) se considera par en este programa? ¿Por qué?

Sí. 0 % 2 = 0, así que cumple numero % 2 === 0 y se suma en sumaPares.

Matemáticamente el cero es par porque es divisible por 2. Si escribes 0, 2, 4, la suma de pares será 6.

8. ¿Para qué sirve el id="formNumeros" en el HTML?

El id identifica de forma única el formulario en la página. JavaScript hace:

document.getElementById('formNumeros')

para obtener ese elemento y registrar el listener submit. Si el id no coincidiera, formNumeros sería null y el script fallaría al intentar addEventListener.

9. ¿Qué hace evento.preventDefault() al enviar el formulario?

Cancela la acción por defecto del navegador al enviar un formulario: normalmente recargar la página.

Gracias a eso el cálculo ocurre en la misma página, el texto del input no se pierde, y el resultado aparece en #resultado sin recargar.

10. ¿Qué comprueba numeros.some((numero) => Number.isNaN(numero))?

some recorre el array y devuelve true si al menos un elemento cumple la condición.

Number.isNaN(numero) es verdadero cuando el valor no es un número válido (por ejemplo si la usuaria escribió 2, hola, 8 → Number("hola") es NaN).

Si hay algún valor inválido, se muestra el mensaje de error y se hace return para no calcular sumas incorrectas.

11. ¿Para qué sirve el atributo required en el input?

Es validación HTML5: el campo no puede enviarse vacío.

Si pulsas el botón sin escribir nada, el navegador muestra su mensaje nativo («Rellena este campo») y no dispara el cálculo hasta que haya texto. Es la primera línea de defensa antes de que JavaScript procese el valor.

12. ¿Por qué el script app.js está al final del body?

Las líneas 1-3 buscan elementos del DOM (formNumeros, numeros, resultado). Esos nodos deben existir ya cuando se ejecuta el script.

Si app.js estuviera en el head sin esperar a DOMContentLoaded, getElementById podría devolver null porque el HTML del formulario aún no se habría creado.

13. Explica las clases de fondo en el body: bg-[url(...)], bg-cover, bg-fixed.

bg-[url('img/wwwatercolor.webp')] — Define la imagen de fondo de la página (acuarela en la carpeta img).

bg-cover — Escala la imagen para cubrir todo el viewport sin dejar huecos.

bg-fixed — El fondo no se mueve al hacer scroll del contenido.

bg-center y bg-no-repeat — Centra la imagen y evita que se repita en mosaico.

14. ¿Qué es innerHTML y qué hace ${sumaPares} en las líneas 28-35?

innerHTML sustituye el contenido HTML interno del div #resultado por el texto que le asignes (puede incluir etiquetas como <p>).

Se usan plantillas literales (backticks `` ` ``). Dentro, ${sumaPares} inserta el valor actual de la variable en ese punto del HTML, igual que concatenar pero más legible.

Así el usuario ve «Suma de pares: 20» con el número ya calculado, sin escribir HTML a mano en el archivo.

15. Si escribes solo abc, 2, 4, ¿qué ocurre y por qué?

Tras split y Number, Number("abc") produce NaN (Not a Number).

numeros.some(... Number.isNaN ...) detecta ese valor inválido, muestra en #resultado:

«Introduce solo números separados por comas.»

y hace return, así que no se calculan sumaPares ni sumaImpares. Es una validación para evitar resultados sin sentido.
