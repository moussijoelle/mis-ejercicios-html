  1. 1 Commencer le fichier : la « boîte » du document

    Comme un cahier : d’abord on écrit !DOCTYPE html, puis on ouvre <html> avec lang="es" (ou fr si ta page est en français). À l’intérieur : head = les réglages invisibles, body = ce qu’on voit à l’écran.

    <!DOCTYPE html>
<html lang="es">
<head>...</head>
<body>...</body>
</html>

  2. 2 Le <head> : mettre le texte en ordre (caractères + mobile)

    charset UTF-8 = les accents (á, ñ, ç) s’affichent bien. viewport = la page s’adapte sur téléphone (largeur = largeur de l’écran).

    <meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>...</title>

  3. 3 La police d’écriture (Google Fonts) — comme choisir un stylo

    On ajoute d’abord preconnect pour aller un peu plus vite, puis le link qui charge Inclusive Sans. Toute la page pourra l’utiliser quand on mettra font-sans sur le body (via @theme à l’étape 5).

  4. 4 Activer Tailwind (CDN navigateur) — le « moteur » des classes

    On charge le script @tailwindcss/browser (v4). Ensuite, les classes du type text-lg deviennent de vrais styles (comme un robot qui remplit le CSS à ta place). Pas besoin d’installer Node pour cet exercice.

    <script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>

  5. 5 Bloc @theme : ta couleur « bordeaux » et la police par défaut

    Dans <style type="text/tailwindcss">, on met @theme : ici --color-burgundy: #901040 crée des classes text-burgundy, border-burgundy, etc. C’est plus propre qu’augmenter 50 fois #901040 partout.

    <style type="text/tailwindcss">
  @theme {
    --font-sans: "Inclusive Sans", ui-sans-serif, system-ui, sans-serif;
    --color-burgundy: #901040;
  }
</style>

  6. 6 Habiller toute la page (body) : gris, hauteur, marge, texte

    Sur body : font-sans = utiliser notre police ; antialiased = lisser un peu les bords des lettres ; bg-gray-100 = grand fond clair (comme un mur) ; min-h-dvh = hauteur au moins = hauteur de l’écran ; py-10 px-4 = air autour ; text-slate-800 = gris de texte par défaut.

    <body class="font-sans antialiased bg-gray-100
  min-h-dvh py-10 px-4 text-slate-800">

  7. 7 <main> : limiter la largeur et centrer

    max-w-2xl = la page ne s’étire pas trop (meilleur pour lire) ; mx-auto = marge horizontale automatique → centré (comme une assiette au milieu de la table).

    <main class="max-w-2xl mx-auto">

  8. 8 Titre + sous-titre

    Le h1 est gros, gras, bordeaux. Le paragraphe d’en dessous : petit, gras, gris foncé, avec un mb-8 (marge en bas) pour aérer avant le formulaire. leading-snug = hauteur de ligne un peu serrée pour un bloc court.

    <h1 class="text-2xl font-bold text-burgundy mb-1">Gastos de la casa</h1>
<p class="text-sm font-bold text-slate-800 mb-8 max-w-xl leading-snug">Ojo: lo que escribes en las cajas lo lee el navegador
como texto; para sumar hace falta pasarlo a número.</p>

  9. 9 Les lignes « label + champ » — grille (grid), pas flex ici

    Pour aligner étiquette et zone de saisie sur 2 colonnes, on a utilisé grid : c’est une table “souple”. grid-cols-[7.5rem_1fr] = première colonne largeur fixe (l’étiquette), 2e colonne prend le reste (1fr = « 1 part du reste »). text-right = texte de l’étiquette collé vers la droite (au bord de la case).

    Flex, c’est quoi (pour comparer) ? C’est plutôt une rangée ou colonne d’objets côte à côte : pratique pour le bouton, la barre de navigation, etc. Ici, la grille a été plus claire car on veut 2 « colonnes imposées » sur toutes les lignes, comme un tableau.

    sm:grid-cols-[8.5rem_1fr] = à partir d’un petit écran, la colonne des libellés est un peu plus large. items-center = centre chaque label et son champ verticalement (sur la hauteur de la ligne).

    class="grid grid-cols-[7.5rem_1fr] ... items-center
  sm:grid-cols-[8.5rem_1fr]"

  10. 10 Style d’un input : bord, coins, ombre, placeholder, focus

    Les champs ont un fond blanc, une bordure grise, un peu d’padding (air à l’intérieur) : px-3 py-2. placeholder:text-rose-300 = le texte indicatif (gris-rose) tant qu’on n’a pas tapé. focus:ring-1 = un petit halo quand on clique dedans, couleur burgundy.

  11. 11 Encadré « Gastos fijos » : bord, position, étiquette qui « chevauche »

    Le cadre a border-2 border-burgundy (grosse bordure), relative sert d’ancre : les enfants en absolute se placent par rapport à cette boîte. Le titre Gastos fijos a absolute -top-3 left-4 : il se colle en haut à gauche, un peu au-dessus de la bordure (d’où l’air en haut du cadre : pt-7). z-10 = il passe devant le trait du cadre.

    space-y-3 (sur le formulaire) et space-y-3 (à l’intérieur) ajoutent de l’espace entre les blocs, comme on empile des cahiers en laissant 1 doigt entre deux.

  12. 12 Où on utilise flex ici

    Dans le JavaScript, on construit chaque ligne du détail en HTML. On a mis class="flex justify-between gap-4 ..." : une rangée flex avec 2 choses (nom de la charge à gauche, chiffre à droite) ; justify-between pousse l’un et l’autre aux bouts (comme aux deux côtés d’une corde). C’est l’idéal dès qu’on veut « gauche / droite » sur une seule ligne, sans forcer 2 colonnes toute la hauteur.

    tabular-nums = chiffres alignés (comme une calculatrice).

    class="flex justify-between gap-4 ... border-b ..."

  13. 13 JavaScript : lire le texte, le transformer en nombre, sommer

    Quand on clique, on lit .value : c’est toujours une chaîne (string) pour un input texte. parseFloat (après replace(",", ".")) donne un nombre, ou on met 0 si c’est illisible. Puis on additionne pour le total, et on injecte le HTML dans #resultado.

    const raw = el.value;  // en un input de texto, .value siempre es una cadena
const num = aNumero(raw);
lineas.push(`<div class="flex justify-between ...">`);

À refaire toi-même : ouvre index.html à côté, et recopie l’idée de chaque étape dans l’ordre. Bon révision !
