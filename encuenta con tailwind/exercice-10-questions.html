Exercice : 10 questions

Español : Preguntas y respuestas para repasar la página « Gastos de la casa ».
Français : Questions et réponses pour réviser (HTML, Tailwind, grille, flex, JS).

  1. Question 1

    [FR] À quoi sert <!DOCTYPE html> en tout début de fichier ?

    [ES] ¿Para qué sirve <!DOCTYPE html> al principio del archivo?

    Réponse / Respuesta

    FR : C’est l’étiquette “mode d’emploi” : elle dit au navigateur « ceci est du HTML5 ». Ainsi le rendu est cohérent (et pas le mode quirks des vieilles pages).

    ES : Indica al navegador que el documento es HTML5, para que use las reglas modernas de visualización (evita el modo de compatibilidad antiguo).

  2. Question 2

    [FR] Que fait charset="UTF-8" dans l’en-tête ?

    [ES] ¿Qué hace charset="UTF-8" en el encabezado?

    Réponse / Respuesta

    FR : UTF-8 sait afficher beaucoup de lettres : accents (é, ñ), symboles (€)… C’est l’encodage du texte, comme l’alphabet de ton fichier.

    ES : Elige cómo se guardan los caracteres: con UTF-8 los acentos y letras raras se ven bien, sin «caracteres raros» (mojibake).

  3. Question 3

    [FR] Pourquoi faut-il name="viewport" sur mobile ?

    [ES] ¿Por qué hace falta name="viewport" en el móvil?

    Réponse / Respuesta

    FR : Sur téléphone, sans cette balise, la page peut paraître trop petite (comme un site d’ordinateur loin d’un microscope). Ici, width=device-width dit : « s’adapte à l’écran qu’on tient ».

    ES : Sin viewport el móvil puede alejar la página. Con width=device-width se usa el ancho real de la pantalla, más cómoda de leer y tocar.

  4. Question 4

    [FR] Qu’apporte le script Tailwind dans le head (version CDN) ?

    [ES] ¿Qué aporta el script de Tailwind en el head (versión CDN)?

    Réponse / Respuesta

    FR : C’est le moteur : il lit les class="..." et génère le style (couleurs, marge, grille, etc.) sans un gros fichier CSS que tu remplis à la main, pour un petit projet ou un exercice — sans installer de logiciels.

    ES : Aplica en el navegador utilidades a partir de las clases (por ejemplo text-lg); no necesitas escribir todo el CSS tú sola, solo combinar nombres cortos.

  5. Question 5

    [FR] Qu’est-ce que @theme { --color-burgundy: #901040; } dans style type="text/tailwindcss" ?

    [ES] ¿Qué es @theme { --color-burgundy: #901040; } en el style de Tailwind?

    Réponse / Respuesta

    FR : C’est ta couleur personnalisée (bordeaux) enregistrée un seul endroit. Après, tu peux utiliser text-burgundy, border-burgundy partout, comme un surnom de la teinte, sans retaper #901040.

    ES : Define un color del tema con un nombre: así puedes usar bg-burgundy, text-burgundy, etc., de forma coherente en toda la página.

  1. Question 6

    [FR] Quelle différence, dans notre page, entre CSS Grid (classes grid…) et Flexbox (flex) ?

    [ES] ¿En qué se distingue en esta página CSS Grid (clases grid) y Flexbox (flex)?

    Réponse / Respuesta

    FR : Grid : on aligne étiquette + zone de saisie en 2 colonnes sur plusieurs lignes (un “petit tableau” souple). Flex : on l’utilise dans le résultat généré par le JS pour placer nom (à gauche) et montant (à droite) sur une seule ligne (justify-between).

    ES : Grid estructura las filas del formulario (etiqueta | input). Flex sirve en las líneas del desglose creadas por JavaScript, para alinear título a la izquierda y cifra a la derecha.

  2. Question 7

    [FR] Pourquoi l’en-tête « Gastos fijos » utilise position: relative (parent) et absolute (l’onglet) ?

    [ES] ¿Por qué el título « Gastos fijos » usa relative en el contenedor y absolute en la etiqueta?

    Réponse / Respuesta

    FR : Le parent relative crée une boîte d’appui : l’enfant absolute se place contre ce bord, pas toute la fenêtre. Comme ça, le petit onglet peut chevaucher proprement le bord haut, avec z-index pour passer devant le trait.

    ES : relative fija el contexto: el título luego se posa con absolute -top-3 respecto a la caja. Sin el padre relativo, absolute se ajustaría a otra caja o al viewport y el efecto de «pegatina en el marco» fallaría.

  3. Question 8

    [FR] Un input (type text) renvoie quel type de valeur en JavaScript element.value ?

    [ES] Un input (tipo text) ¿de qué tipo es element.value en JavaScript?

    Réponse / Respuesta

    FR : C’est toujours une chaîne (string) — comme du texte dans une étiquette. Même si l’usager tape "43" ou "12,5", c’est d’abord du texte ; on convertit avec parseFloat pour calculer, après replace éventuel de , en ..

    ES : Es siempre un string. Los números de los campos de texto no son de tipo number hasta que convertimos con parseFloat o Number (cuidado con comas y vacíos).

  4. Question 9

    [FR] Dans grid-cols-[7.5rem_1fr], que signifie 1fr ?

    [ES] En grid-cols-[7.5rem_1fr], ¿qué significa 1fr?

    Réponse / Respuesta

    FR : fr = fraction de l’espace libre. Ici, la 1ʳᵉ colonne a une largeur fixe (7,5 rem), le reste de l’écran va en 1fr pour le champ (comme 1 part du gâteau de largeur qu’il reste).

    ES : 1fr asigna el espacio restante de la fila a la segunda columna. La etiqueta mantiene un ancho fijo; el input se estira para rellenar el hueco.

  5. Question 10

    [FR] Quel lien faut-il respecter entre label for="..." et l’input pour l’accessibilité ?

    [ES] ¿Qué relación debe existir entre label for y el id del input (accesibilidad)?

    Réponse / Respuesta

    FR : L’attribut for du label doit égaler l’id de l’input associé. Comme un clic d’étiquette = focus dans la bonne case, et les lecteurs d’écran annoncent le rôle (nom du champ) correctement.

    ES : for="nombre" debe coincidir con id="nombre" en el campo. Así al pulsar el texto de la etiqueta, el cursor va al input correcto y se mejora el uso con teclado y tecnologías de apoyo.
