Ir al programa

Verificar número del DNI

Explicaciones, código línea por línea y test de 15 preguntas. Haz clic en cada pregunta para ver la respuesta cuando quieras corregir.

1. En qué consiste el ejercicio

El usuario debe introducir un DNI o NIE con su letra y el programa debe llamar a una función que revise si la letra de control es correcta o no. Cálculo del dígito de control del NIF/NIE (dividir con %).

¿Qué es el DNI?

En España, el DNI (Documento Nacional de Identidad) tiene este formato: 8 cifras + 1 letra. Ejemplo: 12345678Z.

La letra final no es aleatoria. Es un dígito de control: si alguien se equivoca al escribir un número, la letra ya no coincidirá. El programa responde a una pregunta sencilla: «¿La letra que ha escrito la persona es la correcta para esos números?»

Cómo se calcula la letra correcta

Se usa una tabla fija de 23 letras (en este orden):

T R W A G M Y F P D X B N J Z S Q V H L C K E

  1. Se toman las 8 cifras del documento (sin la letra final).
  2. Se divide ese número entre 23.
  3. Se guarda el resto de la división (un número entre 0 y 22). En código eso es el operador %.
  4. Ese resto indica la posición en la tabla de letras.
  5. La letra de esa posición es la letra correcta.

Ejemplo: 12345678 → 12345678 % 23 = 14 → posición 14 → letra Z. Si el usuario escribe 12345678Z → correcto. Si escribe 12345678A → incorrecto.

¿Qué es el NIE?

El NIE (Número de Identidad de Extranjero) empieza por X, Y o Z, luego 7 cifras y una letra de control. Antes del mismo cálculo, se sustituye la primera letra por un dígito:

LetraSe convierte en
X0
Y1
Z2

Ejemplo: X1234567L → se trata como 01234567 y se aplica la misma regla del resto % 23.

Qué hace tu programa

  1. Pide al usuario el DNI o NIE completo (cifras + letra).
  2. Comprueba que el formato sea válido.
  3. Llama a la función esLetraCorrecta para saber si la letra coincide.
  4. Muestra un mensaje claro: válido o no, y cuál sería la letra correcta si falla.
Usuario escribe DNI/NIE
      → validar formato
      → obtener número de 8 cifras (NIE: X/Y/Z → 0/1/2)
      → resto = número % 23
      → letra de la tabla en esa posición
      → comparar con la letra escrita
      → mostrar resultado

6. Test — 15 preguntas

Responde primero en papel o mentalmente. Luego haz clic en cada pregunta para ver la respuesta.

1. ¿Cuántos caracteres tiene un DNI completo?

Un DNI completo tiene 9 caracteres: 8 cifras más 1 letra de control al final. Ejemplo: 12345678Z.

2. ¿Para qué sirve la letra final del DNI?

Es un dígito de control. Se calcula a partir de las cifras con una regla matemática. Si alguien cambia un número por error, la letra ya no cuadra y el programa puede detectarlo.

3. ¿Qué hace el operador % en este ejercicio?

Devuelve el resto de una división. Aquí: numero % 23 da un valor entre 0 y 22 que indica qué letra tomar de la tabla TRWAGMYFPDXBNJZSQVHLCKE. No es el cociente de la división, solo el resto.

4. ¿Por qué se divide entre 23?

Porque la tabla de letras de control del DNI/NIE en España tiene exactamente 23 letras. El resto de dividir entre 23 siempre será un índice válido (de 0 a 22) para elegir una letra de esa tabla.

5. DNI 12345678 — ¿qué letra le corresponde?

12345678 % 23 = 14. En la tabla, la posición 14 (empezando en 0) es la letra Z. Por eso un DNI válido sería 12345678Z.

6. ¿Qué función del programa comprueba si la letra es correcta?

La función esLetraCorrecta(dni). Obtiene la letra que escribió el usuario, calcula la letra correcta con calcularLetraControl y devuelve true o false según coincidan.

7. ¿Qué diferencia hay entre DNI y NIE en este ejercicio?

El DNI empieza por 8 cifras. El NIE empieza por X, Y o Z, luego 7 cifras. Antes del cálculo, X se cambia por 0, Y por 1 y Z por 2, y se forma un número de 8 cifras como en el DNI.

8. NIE que empieza por X — ¿cómo se convierte para calcular?

La X se sustituye por 0. Si el NIE es X1234567L, el número base es 01234567 (8 cifras). Luego se aplica % 23 y la tabla de letras igual que en un DNI.

9. ¿Qué hace valor.slice(-1)?

Devuelve el último carácter del texto. En un DNI es la letra de control que el usuario ha escrito. Ejemplo: en 12345678Z devuelve Z.

10. ¿Para qué sirve esFormatoValido?

Comprueba con expresiones regulares que el texto tenga la forma correcta antes de calcular: o bien 8 dígitos + letra (DNI), o bien X/Y/Z + 7 dígitos + letra (NIE). Si no, muestra error y no intenta calcular.

11. El usuario escribe 12345678A — ¿qué debería pasar?

El formato es válido, pero la letra A no es la correcta. El programa debe mostrar un aviso indicando que para 12345678 la letra debería ser Z.

12. ¿Por qué usamos preventDefault()?

Al enviar un formulario, el navegador suele recargar la página. preventDefault() cancela ese comportamiento para que solo se ejecute nuestro JavaScript y el resultado aparezca en el mismo sitio sin perder lo escrito.

13. ¿Para qué sirven los id en HTML y JS?

El id identifica un elemento de forma única. JavaScript usa getElementById('formDni'), getElementById('dni') y getElementById('resultado') para enlazar el formulario, el input y la zona de mensajes. Si los nombres no coinciden, el programa no funciona.

14. ¿Qué hace LETRAS_CONTROL.charAt(numero % 23)?

Calcula el resto numero % 23 y con ese número obtiene una letra de la cadena LETRAS_CONTROL en esa posición. Es el núcleo del algoritmo oficial de la letra de control.

15. El usuario escribe solo 1234 — ¿qué pasa?

esFormatoValido devuelve false porque no cumple el patrón de 8 cifras + letra ni el de NIE. Se muestra el mensaje de formato no válido en rojo y no se llama a esLetraCorrecta.
