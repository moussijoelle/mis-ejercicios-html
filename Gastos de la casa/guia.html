Guide de l'exercice « Gastos de la casa »

1. De quoi parle cet exercice ? (explication simple)

Bonjour ! Imagine que tu tiens un petit carnet pour noter combien tu dépenses chaque mois pour ta maison : internet, électricité, eau, gaz, loyer… Cet exercice, c'est exactement ça, mais sur une page web.

Ce que tu as construit :

  • Une page web (index.html) avec un formulaire où l'utilisateur peut écrire son nom et ses dépenses.
  • Un programme JavaScript (app.js) qui lit ces informations et calcule le total quand on clique sur le bouton « Calcular Gastos ».

Comment ça marche, en trois étapes :

  1. L'utilisateur remplit les champs (par exemple : luz = 50, agua = 30…).
  2. Il clique sur le bouton.
  3. Le JavaScript additionne tous les nombres et affiche le résultat en bas de la page.

Un point important que tu as déjà noté dans ta page : quand tu lis une valeur depuis le HTML avec JavaScript, elle arrive sous forme de texte (string), même si c'est un nombre. Il faut donc la convertir avec Number() avant de faire des calculs. Sinon, JavaScript pourrait se comporter de façon étrange.

Les deux fichiers travaillent ensemble :

  • index.html → la structure et l'apparence (ce qu'on voit).
  • app.js → la logique (ce qui se passe quand on clique).

4. Test — 15 questions (clique pour voir la réponse)

Réponds d'abord sans regarder. Clique sur chaque question pour révéler la réponse détaillée.

  1. Question 1 — Quel est l'objectif principal de cet exercice ?
    L'objectif est de créer une mini-application web qui permet à un utilisateur de saisir ses dépenses mensuelles du foyer (internet, électricité, eau, gaz, loyer) et d'obtenir automatiquement le total en cliquant sur un bouton. Cet exercice combine trois compétences de base : structurer une page avec HTML, la styliser avec Tailwind CSS, et lui donner de l'intelligence avec JavaScript. C'est une introduction concrète au développement web front-end.
  2. Question 2 — Quelle est la différence entre index.html et app.js ?
    index.html contient la structure visible de la page : titres, champs de saisie, bouton, zone de résultat. C'est le « squelette » et une partie du « décor ». app.js contient la logique : il récupère les éléments de la page, lit leurs valeurs, fait le calcul et affiche le résultat. En résumé : HTML = ce qu'on voit, JavaScript = ce que ça fait. Les deux fichiers sont liés par les attributs id dans le HTML et getElementById dans le JS, ainsi que par la balise <script src="app.js">.
  3. Question 3 — Pourquoi utilise-t-on l'attribut id sur les inputs ?
    L'id est un identifiant unique pour chaque élément HTML. JavaScript en a besoin pour « retrouver » un élément précis dans la page grâce à document.getElementById("…"). Sans id, le script ne saurait pas quel champ correspond à « internet », « luz », etc. L'id sert aussi à lier un <label> à son champ via for="nomDuId" : quand on clique sur le label, le curseur se place automatiquement dans l'input associé.
  4. Question 4 — Que fait document.getElementById() ?
    C'est une méthode JavaScript qui parcourt le document HTML et retourne l'élément dont l'id correspond exactement à la chaîne passée entre parenthèses. Par exemple, document.getElementById("luz") retourne l'input avec id="luz". Une fois l'élément récupéré, on peut lire sa valeur avec .value ou modifier son texte avec .textContent. C'est le pont principal entre le HTML statique et le JavaScript dynamique.
  5. Question 5 — Pourquoi faut-il utiliser Number() avant d'additionner les valeurs ?
    Parce que .value d'un input renvoie toujours une chaîne de caractères (string), même pour un type="number". Si tu fais "10" + "20" en JavaScript sans conversion, tu obtiens "1020" (concaténation de textes), pas 30. Number("10") transforme le texte en vrai nombre, ce qui permet l'addition mathématique. Si le champ est vide, Number("") donne 0, ce qui évite une erreur dans le calcul.
  6. Question 6 — Que se passe-t-il quand l'utilisateur clique sur « Calcular Gastos » ?
    Le clic déclenche l'attribut onclick="calcular()" sur le bouton. Le navigateur cherche alors la fonction globale calcular définie dans app.js et l'exécute. La fonction lit les valeurs actuelles des cinq champs de dépenses, les convertit en nombres, les additionne, puis écrit le résultat formaté (Total: X €) dans l'élément totalGastos. La page ne se recharge pas grâce à type="button".
  7. Question 7 — Pourquoi le bouton a-t-il type="button" et non type="submit" ?
    Un bouton type="submit" à l'intérieur d'un <form> envoie le formulaire au serveur et recharge généralement la page. Ici, on veut juste lancer un calcul en JavaScript, sans rechargement. type="button" indique au navigateur : « ce bouton ne soumet pas le formulaire ». Seul le code JavaScript (onclick="calcular()") décide de ce qui se passe au clic.
  8. Question 8 — À quoi sert la balise <fieldset> dans ce formulaire ?
    <fieldset> regroupe plusieurs champs qui appartiennent à la même catégorie. Ici, tous les « gastos fijos » (dépenses fixes) sont visuellement encadrés ensemble. La balise <legend> à l'intérieur donne un titre au groupe (« Gastos fijos »). C'est utile pour l'organisation visuelle et aussi pour l'accessibilité : les lecteurs d'écran comprennent que ces champs forment un ensemble logique.
  9. Question 9 — Quelle est la différence entre type="text" et type="number" ?
    type="text" accepte n'importe quel texte (lettres, chiffres, symboles) — utilisé pour le nom et les apellidos. type="number" est réservé aux valeurs numériques : le navigateur peut afficher des flèches pour augmenter/diminuer, et sur mobile, proposer un clavier numérique. Attention : même avec type="number", JavaScript reçoit la valeur comme string via .value, d'où la nécessité de Number().
  10. Question 10 — Pourquoi le script app.js est-il placé à la fin du body ?
    Parce que le JavaScript a besoin que les éléments HTML existent déjà dans la page avant de les chercher avec getElementById. Si le script était chargé en haut, avant les inputs, les éléments n'existeraient pas encore et les variables seraient null. En plaçant <script src="app.js"> juste avant </body>, on garantit que tout le HTML est chargé quand le script s'exécute.
  11. Question 11 — Que fait console.log(internet.value) ?
    Cette ligne affiche la valeur actuelle du champ internet dans la console de développement du navigateur (accessible avec F12 → onglet Console). Ce n'est pas visible pour l'utilisateur final sur la page. C'est un outil de débogage : le développeur vérifie que le JavaScript lit bien la bonne valeur. Par exemple, si internet a value="43", la console affichera 43.
  12. Question 12 — Qu'est-ce que Tailwind CSS et comment est-il utilisé ici ?
    Tailwind CSS est un framework qui permet de styliser une page en ajoutant des classes directement dans le HTML, sans écrire beaucoup de CSS classique. Par exemple, flex justify-center centre le contenu, bg-gray-100 met un fond gris clair, text-pink-800 colore le texte en rose foncé. Dans cet exercice, Tailwind est chargé via CDN (internet). Le bloc <style type="text/tailwindcss"> permet aussi de personnaliser la police et quelques styles globaux.
  13. Question 13 — Pourquoi le formulaire a-t-il class="w-3/4" ?
    w-3/4 est une classe Tailwind qui signifie « width: 75% » (three quarters = trois quarts). Le formulaire occupe donc les trois quarts de la largeur de l'écran, laissant un peu d'espace sur les côtés. Combiné avec flex justify-center sur le body, cela centre un formulaire de taille raisonnable, ni trop étroit ni trop large. C'est un choix esthétique et de lisibilité.
  14. Question 14 — Que signifie totalGastos.textContent = `Total: ${total} €` ?
    Cette ligne modifie le texte visible à l'intérieur de l'élément HTML totalGastos. .textContent remplace tout le contenu textuel du paragraphe. Les backticks `` ` `` créent un template literal : ${total} est remplacé par la valeur de la variable total au moment de l'exécution. Si le total vaut 123, l'utilisateur verra « Total: 123 € » s'afficher dans la zone en bas du formulaire.
  15. Question 15 — Pourquoi les champs nombre et apellidos sont-ils récupérés en JS mais pas utilisés dans calcular() ?
    Actuellement, calcular() ne somme que les dépenses (internet, luz, agua, gas, alquiler). Les champs nombre et apellidos sont déjà connectés au JavaScript (lignes 1-2 de app.js), ce qui prépare une évolution future : par exemple afficher « Bonjour María, ton total est de 250 € ». Les récupérer dès le début est une bonne pratique — on établit le lien HTML/JS — même si la logique métier ne les utilise pas encore. C'est une base prête à être enrichie.
